# 1. 탐색(Search)이란?

탐색은 원하는 데이터를 효율적으로 찾는 과정이다.

``` text
데이터 집합
     │
 Search
     │
원하는 데이터
```

대표 알고리즘

-   Sequential Search
-   Binary Search
-   DFS
-   BFS

------------------------------------------------------------------------

# 2. Sequential Search

앞에서부터 하나씩 비교한다.

``` text
[3][7][9][2][8]

찾기 : 2
 ↓ ↓ ↓ ✓
```

-   정렬 불필요
-   평균 O(n)

장점 - 구현이 매우 쉽다.

단점 - 데이터가 많으면 느리다.

------------------------------------------------------------------------

# 3. Binary Search

반씩 나누며 탐색한다.

반드시 **정렬된 데이터**가 필요하다.

``` text
1 3 5 7 9 11 13

찾기 : 9

중간=7
      ↓
오른쪽
      ↓
9 발견
```

시간복잡도

    O(log n)

------------------------------------------------------------------------

# 4. DFS(Depth First Search)

깊이 우선 탐색

자료구조

-   Stack
-   Recursion

``` text
A
├─B
│ ├─D
│ └─E
└─C

DFS

A → B → D → E → C
```

활용

-   미로 탐색
-   백트래킹
-   파일 시스템

------------------------------------------------------------------------

# 5. BFS(Breadth First Search)

너비 우선 탐색

자료구조

-   Queue

``` text
A
├─B
├─C
├─D

BFS

A → B → C → D
```

활용

-   최단 경로
-   SNS 친구 추천
-   네트워크 탐색

------------------------------------------------------------------------

# 6. DFS와 BFS 비교

  항목        DFS          BFS
  ----------- ------------ --------------------------
  자료구조    Stack        Queue
  탐색 방식   깊이 우선    너비 우선
  최단경로    보장 안 함   보장(가중치 없는 그래프)
  메모리      적음         많음

------------------------------------------------------------------------

# 7. Recursion

함수가 자기 자신을 호출하는 방식이다.

``` text
factorial(4)
 ↓
factorial(3)
 ↓
factorial(2)
 ↓
factorial(1)
```

주의사항

-   종료 조건(Base Case) 필수
-   Stack Overflow 가능

------------------------------------------------------------------------

# 8. 시간복잡도 비교

  알고리즘            시간복잡도
  ------------------- ------------
  Sequential Search   O(n)
  Binary Search       O(log n)
  DFS                 O(V+E)
  BFS                 O(V+E)

# 예상 문제

## 문제 1

Binary Search의 전제 조건은?

1.  Queue 사용
2.  데이터가 정렬되어 있어야 한다.
3.  Graph만 가능
4.  Stack만 사용

**정답:** 2

### 상세 해설

Binary Search는 중간값을 기준으로 탐색하므로 반드시 정렬된 데이터에서만
사용할 수 있다.

------------------------------------------------------------------------

## 문제 2

DFS에서 일반적으로 사용하는 자료구조는?

1.  Queue
2.  Hash
3.  Stack
4.  Heap

**정답:** 3

### 상세 해설

DFS는 Stack 또는 재귀 호출을 이용하여 깊이 우선으로 탐색한다.

------------------------------------------------------------------------

## 문제 3

최단 경로 탐색(가중치 없음)에 적합한 알고리즘은?

1.  DFS
2.  BFS
3.  Bubble Sort
4.  Selection Sort

**정답:** 2

### 상세 해설

BFS는 가까운 노드부터 탐색하므로 가중치 없는 그래프에서 최단 경로를
보장한다.

------------------------------------------------------------------------

## 문제 4

Sequential Search의 평균 시간복잡도는?

1.  O(1)
2.  O(log n)
3.  O(n)
4.  O(n²)

**정답:** 3

### 상세 해설

최악과 평균 모두 선형 탐색이므로 O(n)이다.

------------------------------------------------------------------------

## 문제 5

재귀 함수 작성 시 가장 중요한 것은?

1.  Index 생성
2.  종료 조건(Base Case)
3.  Queue 생성
4.  Pivot 선택

**정답:** 2

### 상세 해설

종료 조건이 없으면 함수가 무한 호출되어 Stack Overflow가 발생한다.

------------------------------------------------------------------------

# 오늘의 핵심 요약

-   Sequential = O(n)
-   Binary = O(log n), 정렬 필수
-   DFS = Stack / Recursion
-   BFS = Queue
-   BFS = 최단 경로(가중치 없음)
-   Recursion = 종료 조건 필수
