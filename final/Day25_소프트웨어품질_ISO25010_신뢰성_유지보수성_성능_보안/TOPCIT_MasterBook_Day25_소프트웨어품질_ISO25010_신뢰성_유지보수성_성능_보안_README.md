# 1. 소프트웨어 품질이란?

소프트웨어 품질은 사용자의 요구사항과 기대를 얼마나 만족하는지를
나타내는 정도이다.

``` text
요구사항
   ↓
설계
   ↓
개발
   ↓
테스트
   ↓
품질 확보
```

품질은 테스트만으로 확보되지 않으며 개발 전 과정에서 관리해야 한다.

------------------------------------------------------------------------

# 2. ISO/IEC 25010 품질 모델

  품질 특성                설명
  ------------------------ -------------
  Functional Suitability   기능 적합성
  Performance Efficiency   성능 효율성
  Compatibility            호환성
  Usability                사용성
  Reliability              신뢰성
  Security                 보안성
  Maintainability          유지보수성
  Portability              이식성

------------------------------------------------------------------------

# 3. 주요 품질 특성

## 기능 적합성

-   요구사항 충족
-   정확성
-   완전성

## 성능 효율성

-   응답시간
-   처리량
-   자원 사용률

## 신뢰성

-   장애 없이 동작
-   가용성
-   장애 복구

## 유지보수성

-   수정 용이성
-   모듈화
-   재사용성

## 보안성

-   인증(Authentication)
-   권한(Authorization)
-   기밀성
-   무결성
-   감사 로그

------------------------------------------------------------------------

# 4. 품질 지표

  지표             의미
  ---------------- ----------------
  MTBF             평균 고장 간격
  MTTR             평균 복구 시간
  Availability     가용성
  Defect Density   결함 밀도

가용성 계산

``` text
Availability = MTBF / (MTBF + MTTR)
```

# 예상 문제

## 문제 1

ISO/IEC 25010의 품질 특성이 아닌 것은?

1.  Reliability
2.  Maintainability
3.  Security
4.  RAID

**정답:** 4

### 상세 해설

RAID는 저장장치 기술이며 ISO/IEC 25010 품질 특성이 아니다.

------------------------------------------------------------------------

## 문제 2

평균 복구 시간을 의미하는 것은?

1.  MTBF
2.  MTTR
3.  TPS
4.  SLA

**정답:** 2

### 상세 해설

MTTR(Mean Time To Repair)은 장애 발생 후 정상 상태까지 복구하는 평균
시간을 의미한다.

------------------------------------------------------------------------

## 문제 3

응답시간과 처리량은 어떤 품질 특성에 속하는가?

1.  Security
2.  Performance Efficiency
3.  Portability
4.  Usability

**정답:** 2

### 상세 해설

성능 효율성은 응답시간, 처리량, 자원 사용률 등을 평가한다.

------------------------------------------------------------------------

## 문제 4

코드 품질을 지속적으로 점검하는 대표 도구는?

1.  SonarQube
2.  Bubble Sort
3.  Redis
4.  Apache

**정답:** 1

### 상세 해설

SonarQube는 정적 분석을 통해 버그, 코드 스멜, 보안 취약점을 점검한다.

------------------------------------------------------------------------

## 문제 5

Availability 계산식으로 올바른 것은?

1.  MTTR / MTBF
2.  MTBF / (MTBF + MTTR)
3.  MTBF × MTTR
4.  MTTR - MTBF

**정답:** 2

### 상세 해설

MTBF가 길고 MTTR이 짧을수록 시스템 가용성이 높아진다.

------------------------------------------------------------------------

# 오늘의 핵심 요약

-   품질은 개발 전 과정에서 관리
-   ISO/IEC 25010 = 8가지 품질 특성
-   MTBF = 평균 고장 간격
-   MTTR = 평균 복구 시간
-   Availability = MTBF / (MTBF + MTTR)
-   SonarQube = 코드 품질 분석
