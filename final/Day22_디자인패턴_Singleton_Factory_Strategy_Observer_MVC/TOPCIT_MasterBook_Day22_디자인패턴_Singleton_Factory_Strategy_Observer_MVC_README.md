# 1. 디자인 패턴이란?

디자인 패턴은 자주 발생하는 설계 문제를 해결하기 위한 검증된 설계
방법이다.

장점

-   유지보수 향상
-   재사용성 증가
-   결합도 감소
-   협업 용이

------------------------------------------------------------------------

# 2. 디자인 패턴 분류

  분류               목적           예
  ------------------ -------------- --------------------
  생성(Creational)   객체 생성      Singleton, Factory
  구조(Structural)   객체 연결      Adapter, Decorator
  행위(Behavioral)   객체 간 동작   Strategy, Observer

------------------------------------------------------------------------

# 3. Singleton Pattern

애플리케이션에서 객체를 하나만 생성한다.

``` text
Application
    │
 Singleton Instance
```

활용

-   Logger
-   설정(Configuration)
-   Cache Manager

장점 - 메모리 절약 - 일관성 유지

주의 - 동시성(Thread Safety) 고려

------------------------------------------------------------------------

# 4. Factory Pattern

객체 생성을 전담하는 패턴이다.

``` text
Client
  │
Factory
 ├─ Car
 ├─ Bus
 └─ Truck
```

장점

-   생성 로직 분리
-   결합도 감소
-   확장성 향상

------------------------------------------------------------------------

# 5. Strategy Pattern

알고리즘을 객체로 분리하여 런타임에 교체한다.

예)

-   결제 방식
-   압축 방식
-   정렬 방식

``` text
Context
 ├─ Card Strategy
 ├─ Cash Strategy
 └─ Point Strategy
```

------------------------------------------------------------------------

# 6. Observer Pattern

상태가 변경되면 등록된 객체들에게 자동으로 알린다.

``` text
Subject
 ├─ Observer A
 ├─ Observer B
 └─ Observer C
```

활용

-   이벤트 시스템
-   GUI
-   Pub/Sub

------------------------------------------------------------------------

# 7. MVC(Model-View-Controller)

  구성         역할
  ------------ -----------
  Model        데이터
  View         화면
  Controller   요청 처리

``` text
User
 ↓
Controller
 ↓
Model
 ↓
View
```

------------------------------------------------------------------------

# 8. Spring Framework 활용

-   Bean Factory → Factory Pattern
-   ApplicationContext → Singleton
-   Spring Event → Observer
-   DI → Strategy 활용
-   Spring MVC → MVC

# 예상 문제

## 문제 1

애플리케이션에서 객체를 하나만 생성하는 패턴은?

1.  Strategy
2.  Singleton
3.  Observer
4.  MVC

**정답:** 2

### 해설

Singleton은 하나의 인스턴스를 공유하여 메모리 사용을 줄이고 일관성을
유지한다.

------------------------------------------------------------------------

## 문제 2

객체 생성을 담당하는 패턴은?

1.  Factory
2.  MVC
3.  Queue
4.  Stack

**정답:** 1

### 해설

Factory는 생성 로직을 캡슐화하여 결합도를 낮춘다.

------------------------------------------------------------------------

## 문제 3

이벤트 기반 알림에 적합한 패턴은?

1.  Observer
2.  Singleton
3.  Adapter
4.  Heap

**정답:** 1

### 해설

Observer는 Subject의 상태 변경을 Observer에게 자동으로 전달한다.

------------------------------------------------------------------------

## 문제 4

Spring MVC의 Controller 역할은?

1.  화면 출력
2.  데이터 저장
3.  요청 처리 및 흐름 제어
4.  DB 백업

**정답:** 3

### 해설

Controller는 요청을 받아 비즈니스 로직을 호출하고 View를 선택한다.

------------------------------------------------------------------------

## 문제 5

전략을 런타임에 교체하기 적합한 패턴은?

1.  Strategy
2.  Singleton
3.  Factory
4.  Bridge

**정답:** 1

### 해설

Strategy는 알고리즘을 캡슐화하여 쉽게 교체할 수 있다.
