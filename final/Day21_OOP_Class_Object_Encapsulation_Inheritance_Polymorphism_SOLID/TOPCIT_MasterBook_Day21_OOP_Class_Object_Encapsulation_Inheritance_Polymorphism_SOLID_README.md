# 1. 객체지향 프로그래밍(OOP)

객체(Object)를 중심으로 프로그램을 설계하는 개발 패러다임이다.

``` text
현실 세계
   │
객체(Object)
   │
Class
   │
Program
```

대표 언어

-   Java
-   C++
-   C#
-   Python(지원)

------------------------------------------------------------------------

# 2. 클래스(Class)와 객체(Object)

## Class

객체를 만들기 위한 설계도

## Object

클래스로부터 생성된 실제 인스턴스

예)

``` java
class Car{
   String color;
}

Car myCar = new Car();
```

------------------------------------------------------------------------

# 3. 객체지향 4대 특징

## ① 캡슐화(Encapsulation)

데이터와 메서드를 하나로 묶고 외부 접근을 제한한다.

장점

-   정보 은닉
-   유지보수 향상

------------------------------------------------------------------------

## ② 상속(Inheritance)

기존 클래스를 재사용하여 새로운 클래스를 생성한다.

``` text
Animal
  │
 ├── Dog
 └── Cat
```

장점

-   코드 재사용
-   확장성

------------------------------------------------------------------------

## ③ 다형성(Polymorphism)

동일한 인터페이스로 다양한 객체를 처리한다.

예)

``` java
Animal a=new Dog();
Animal b=new Cat();
```

------------------------------------------------------------------------

## ④ 추상화(Abstraction)

불필요한 세부 구현을 숨기고 핵심 기능만 표현한다.

예)

-   Interface
-   Abstract Class

------------------------------------------------------------------------

# 4. Interface와 Abstract Class

  항목          Interface      Abstract
  ------------- -------------- ----------
  다중 구현     가능           불가
  일반 메서드   가능(Java8+)   가능
  생성자        없음           있음

------------------------------------------------------------------------

# 5. SOLID 원칙

  원칙   의미
  ------ -----------------
  S      단일 책임
  O      개방-폐쇄
  L      리스코프 치환
  I      인터페이스 분리
  D      의존성 역전

실무에서는 유지보수성과 테스트 용이성을 높이기 위해 중요하게 사용된다.

# PM Note

객체지향이 잘 적용된 시스템은

-   유지보수 용이
-   테스트 용이

# 예상 문제

## 문제 1

객체를 생성하기 위한 설계도는?

1.  Method
2.  Class
3.  Package
4.  Library

**정답:** 2

### 상세 해설

Class는 객체(Object)를 생성하기 위한 템플릿이며 Object는 Class로부터
생성된 인스턴스이다.

------------------------------------------------------------------------

## 문제 2

객체지향의 4대 특징이 아닌 것은?

1.  캡슐화
2.  상속
3.  다형성
4.  정렬

**정답:** 4

### 상세 해설

정렬은 알고리즘 개념이며 객체지향의 핵심 특징은 캡슐화, 상속, 다형성,
추상화이다.

------------------------------------------------------------------------

## 문제 3

정보 은닉을 제공하는 객체지향 특징은?

1.  상속
2.  추상화
3.  캡슐화
4.  다형성

**정답:** 3

### 상세 해설

캡슐화는 내부 구현을 숨기고 필요한 기능만 외부에 제공한다.

------------------------------------------------------------------------

## 문제 4

SOLID에서 D는 무엇을 의미하는가?

1.  Data
2.  Dependency Inversion
3.  Database
4.  Driver

**정답:** 2

### 상세 해설

Dependency Inversion Principle은 추상화에 의존하도록 설계하여 결합도를
낮춘다.

------------------------------------------------------------------------

## 문제 5

다형성의 예로 가장 적절한 것은?

1.  Animal a = new Dog();
2.  int a = 10;
3.  SELECT \* FROM EMP;
4.  RAID5

**정답:** 1

### 상세 해설

부모 타입으로 다양한 자식 객체를 참조하는 것이 다형성의 대표적인 예이다.
