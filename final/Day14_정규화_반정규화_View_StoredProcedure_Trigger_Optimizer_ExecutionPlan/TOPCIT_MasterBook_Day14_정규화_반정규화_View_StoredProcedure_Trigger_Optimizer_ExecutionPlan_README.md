# 1. 정규화(Normalization)

정규화는 데이터 중복을 최소화하고 무결성을 높이기 위한 설계 기법이다.

## 장점

-   데이터 중복 제거
-   이상현상(삽입/수정/삭제) 방지
-   유지보수 향상

## 정규형

  단계   핵심
  ------ ---------------------
  1NF    원자값
  2NF    부분 함수 종속 제거
  3NF    이행 함수 종속 제거
  BCNF   결정자 후보키

------------------------------------------------------------------------

# 2. 반정규화(Denormalization)

성능 향상을 위해 의도적으로 중복을 허용하는 기법이다.

장점 - JOIN 감소 - 조회 성능 향상

단점 - 데이터 중복 - UPDATE 비용 증가

실무에서는 조회가 매우 많은 시스템에서 사용한다.

------------------------------------------------------------------------

# 3. View

View는 SELECT 결과를 논리적인 테이블처럼 제공한다.

``` sql
CREATE VIEW vw_employee AS
SELECT emp_id,name
FROM employee;
```

장점 - 보안 - SQL 단순화 - 재사용

------------------------------------------------------------------------

# 4. Stored Procedure

여러 SQL을 하나의 프로그램처럼 실행하는 객체이다.

``` sql
CREATE PROCEDURE proc_bonus()
BEGIN
 UPDATE employee
 SET bonus=bonus*1.1;
END;
```

장점 - 네트워크 트래픽 감소 - 로직 재사용 - 성능 향상

------------------------------------------------------------------------

# 5. Trigger

특정 이벤트 발생 시 자동 실행되는 객체이다.

``` text
INSERT
 UPDATE
 DELETE
      │
 Trigger
```

활용 - 감사 로그 - 변경 이력 - 자동 계산

주의: 과도한 Trigger는 성능 저하를 유발할 수 있다.

------------------------------------------------------------------------

# 6. Optimizer

Optimizer는 가장 효율적인 SQL 실행 계획을 선택한다.

판단 요소 - 통계 정보 - Index - 데이터 분포 - 비용(Cost)

------------------------------------------------------------------------

# 7. Execution Plan

Execution Plan은 SQL 실행 과정을 보여준다.

주요 확인 항목

  항목              의미
  ----------------- ------------------
  Full Table Scan   전체 테이블 조회
  Index Scan        인덱스 사용
  Nested Loop       작은 데이터 JOIN
  Hash Join         대용량 JOIN
  Sort              정렬 작업

------------------------------------------------------------------------

# 8. SQL 튜닝 기본

튜닝 순서

``` text
Slow Query
   ↓
Execution Plan
   ↓
Index 확인
   ↓
Statistics
   ↓
Rewrite SQL
```

튜닝 포인트 - SELECT \* - 불필요한 ORDER BY - 함수 사용으로 인덱스
미사용 - LIKE '%abc'

# 예상 문제

## 문제 1

정규화의 가장 큰 목적은?

1.  CPU 성능 향상
2.  데이터 중복 제거
3.  RAID 구성
4.  네트워크 속도 향상

**정답:** 2

### 상세 해설

정규화는 데이터 중복을 줄이고 데이터 무결성을 확보하기 위한 설계
기법이다.

------------------------------------------------------------------------

## 문제 2

조회 성능 향상을 위해 의도적으로 중복을 허용하는 기법은?

1.  정규화
2.  반정규화
3.  트랜잭션
4.  샤딩

**정답:** 2

### 상세 해설

반정규화는 JOIN을 줄여 조회 성능을 높이지만 중복 데이터 관리 비용이
증가한다.

------------------------------------------------------------------------

## 문제 3

SQL 실행 계획을 선택하는 구성 요소는?

1.  Optimizer
2.  Trigger
3.  View
4.  Cursor

**정답:** 1

### 상세 해설

Optimizer는 비용 기반(CBO) 또는 규칙 기반(RBO) 방식으로 최적 실행 계획을
선택한다.

------------------------------------------------------------------------

## 문제 4

자동으로 실행되는 DB 객체는?

1.  View
2.  Trigger
3.  Index
4.  Sequence

**정답:** 2

### 상세 해설

Trigger는 INSERT, UPDATE, DELETE 이벤트 발생 시 자동으로 실행된다.

------------------------------------------------------------------------

## 문제 5

다음 중 인덱스를 효율적으로 사용하기 어려운 SQL은?

1.  WHERE emp_id=100
2.  WHERE name LIKE 'Kim%'
3.  WHERE name LIKE '%Kim'
4.  WHERE dept_id=10

**정답:** 3

### 상세 해설

앞에 와일드카드('%')가 오면 일반 B+Tree 인덱스를 활용하기 어려워 Full
Table Scan이 발생할 가능성이 높다.
