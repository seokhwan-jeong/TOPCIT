> 목표: TOPCIT 500점 이상

## 학습 목표

-   데이터베이스 핵심 개념을 정리한다.
-   AI/LLM/RAG의 차이와 활용을 이해한다.
-   DevSecOps 보안 도구를 구분한다.

------------------------------------------------------------------------

# 핵심 요약

## 데이터베이스

-   정규화: 중복 제거, 무결성 향상
-   인덱스: 조회 성능 향상(과도하면 쓰기 성능 저하)
-   ACID: Atomicity, Consistency, Isolation, Durability
-   MVCC: 읽기/쓰기 동시성 향상

## AI

-   LLM: 대규모 언어모델
-   RAG: 검색 + 생성
-   Embedding: 의미를 벡터로 표현
-   Vector DB: 유사도 검색

## DevSecOps

-   SAST: 정적 분석
-   DAST: 동적 분석
-   SCA: 오픈소스 의존성 분석
-   SBOM: 소프트웨어 구성요소 목록
-   Trivy: 컨테이너 이미지 취약점 스캔

------------------------------------------------------------------------

# 실전 문제 (20문항)

1.  정규화의 주요 목적은? ① 중복 제거 ② 속도 저하 ③ 암호화 ④ 백업 정답:①

2.  인덱스의 주목적은? ① 조회 성능 향상 ② 저장공간 절감 ③ 로그 관리 ④
    암호화 정답:①

3.  ACID의 D는? ① Durability ② Data ③ Distribution ④ Dependency 정답:①

4.  MVCC의 장점은? ① 동시성 향상 ② 용량 감소 ③ 암호화 ④ 백업 정답:①

5.  LLM은? ① 대규모 언어모델 ② DB엔진 ③ OS ④ API Gateway 정답:①

6.  RAG의 목적은? ① 최신 정보 기반 응답 ② GPU 절감 ③ 로그 압축 ④ 백업
    정답:①

7.  Embedding은? ① 텍스트를 벡터화 ② 암호화 ③ 압축 ④ 백업 정답:①

8.  Vector DB 용도는? ① 유사도 검색 ② DNS ③ RAID ④ NAT 정답:①

9.  SAST는? ① 소스코드 정적 분석 ② 동적 분석 ③ 이미지 스캔 ④ 로그 분석
    정답:①

10. DAST는? ① 실행 중 보안 테스트 ② 정적 분석 ③ 백업 ④ 컴파일 정답:①

11. SCA는? ① 오픈소스 취약점 분석 ② DB 튜닝 ③ 암호화 ④ 라우팅 정답:①

12. SBOM의 목적은? ① 구성요소 관리 ② DB 관리 ③ VPN ④ 스토리지 정답:①

13. Trivy는? ① 이미지 취약점 검사 ② 모니터링 ③ 컴파일 ④ DNS 정답:①

14. SQL 튜닝 시 먼저 확인할 것은? ① 실행 계획 ② RAID ③ BIOS ④ VLAN
    정답:①

15. OLTP 특징은? ① 짧은 트랜잭션 ② 대량 집계 ③ 배치 전용 ④ 분석 전용
    정답:①

16. OLAP 특징은? ① 분석 중심 ② 실시간 거래 ③ 인증 ④ DHCP 정답:①

17. RAG 구성요소가 아닌 것은? ① Retriever ② Generator ③ Vector DB ④ RAID
    정답:④

18. 공급망 보안 강화에 가장 관련 있는 것은? ① SBOM ② FTP ③ ICMP ④ NAT
    정답:①

19. AI 환각 감소에 효과적인 것은? ① RAG ② RAM 증설 ③ RAID ④ CPU 증설
    정답:①

20. DevSecOps의 핵심은? ① 개발 초기부터 보안 통합 ② 운영 후 보안 ③
    테스트 생략 ④ 수동 배포 정답:①
