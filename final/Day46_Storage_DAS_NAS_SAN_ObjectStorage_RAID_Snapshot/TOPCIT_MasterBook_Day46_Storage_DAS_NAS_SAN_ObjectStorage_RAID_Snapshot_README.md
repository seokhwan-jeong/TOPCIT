# 1. 스토리지 종류

  종류     연결 방식        특징            활용
  -------- ---------------- --------------- ----------------
  DAS      서버 직접 연결   단순, 저비용    단일 서버
  NAS      Ethernet         파일 공유       파일 서버
  SAN      FC/iSCSI         Block Storage   DB, 가상화
  Object   HTTP/S3          대용량 객체     백업, AI, 로그

------------------------------------------------------------------------

# 2. SAN vs NAS

  항목        SAN          NAS
  ----------- ------------ -----------
  접근        Block        File
  프로토콜    FC, iSCSI    NFS, SMB
  성능        매우 높음    높음
  대표 용도   DB, VMware   파일 공유

------------------------------------------------------------------------

# 3. RAID

  RAID     특징              장애 허용
  -------- ----------------- -----------
  RAID0    Striping          0
  RAID1    Mirroring         1
  RAID5    Striping+Parity   1
  RAID6    Dual Parity       2
  RAID10   1+0               1쌍 이상

예시: 1TB 디스크 4개 - RAID0 = 4TB - RAID1 = 2TB - RAID5 = 3TB - RAID6 =
2TB - RAID10 = 2TB

------------------------------------------------------------------------

# 4. Snapshot vs Backup

Snapshot - 시점 복구 - 빠름 - 동일 스토리지 의존

Backup - 별도 저장 - 재해 복구 가능 - 복구 시간 상대적으로 길다.

------------------------------------------------------------------------

# 5. Deduplication / Compression

Deduplication - 중복 블록 제거 - 백업 용량 절감

Compression - 데이터 압축 - 저장 공간 절약

------------------------------------------------------------------------

# 6. Thin Provisioning

논리 용량을 실제보다 크게 할당하고 필요 시 물리 공간을 사용한다.

장점 - 저장 공간 효율 - 초기 비용 절감

주의 - Oversubscription 관리 필요

# TOPCIT 실전 문제

## 문제1 (중 \| ★★★★★)

DB 서버에 가장 적합한 스토리지는?

① NAS ② SAN ③ Object Storage ④ Tape

**정답: ②**

**해설:** SAN은 Block 단위 접근과 높은 IOPS를 제공해 DB에 적합하다.

------------------------------------------------------------------------

## 문제2 (상 \| ★★★★★)

1TB 디스크 6개를 RAID5로 구성하면 사용 가능한 용량은?

① 4TB ② 5TB ③ 6TB ④ 3TB

**정답: ②**

**해설:** RAID5 용량 = (N-1) × 디스크 용량 = (6-1)×1TB = 5TB.

------------------------------------------------------------------------

## 문제3 (상 \| ★★★★★)

Snapshot의 가장 큰 특징은?

① 별도 장비에 장기 보관 ② 특정 시점의 빠른 복구 ③ 테이프 백업 전용 ④
반드시 다른 센터 저장

**정답: ②**

------------------------------------------------------------------------

## 문제4 (시나리오 \| ★★★★★)

대용량 이미지와 AI 학습 데이터를 저비용으로 저장하려면?

① DAS ② SAN ③ Object Storage ④ RAID1

**정답: ③**

**해설:** Object Storage는 확장성과 비용 효율성이 뛰어나 대용량 비정형
데이터에 적합하다.

------------------------------------------------------------------------

## 문제5 (시나리오 \| ★★★★★)

DR 구축 시 RPO를 최소화하려면 가장 적절한 기술은?

① Snapshot만 수행 ② 동기 스토리지 복제 ③ RAID0 ④ Compression

**정답: ②**

**해설:** 동기 복제는 데이터 손실을 거의 없애 RPO를 최소화한다.
