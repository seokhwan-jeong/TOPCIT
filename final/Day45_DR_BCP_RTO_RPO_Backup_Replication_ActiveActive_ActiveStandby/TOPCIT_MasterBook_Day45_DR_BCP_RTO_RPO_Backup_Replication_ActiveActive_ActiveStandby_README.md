# 1. DR과 BCP

**DR(Disaster Recovery)** 는 장애 발생 후 IT 서비스를 복구하는 활동이다.

**BCP(Business Continuity Planning)** 는 재해 상황에서도 핵심 업무를
지속하기 위한 조직 전체의 계획이다.

BCP ⊃ DR

------------------------------------------------------------------------

# 2. RTO / RPO

  용어   의미
  ------ -----------------------
  RTO    서비스 복구 목표 시간
  RPO    데이터 손실 허용 시점

예시

-   RTO = 2시간
-   RPO = 10분

⇒ 장애 후 2시간 내 복구, 최대 10분 데이터 손실 허용

------------------------------------------------------------------------

# 3. DR 구성 방식

  방식             특징                비용
  ---------------- ------------------- -----------
  Active-Active    두 센터 모두 운영   매우 높음
  Active-Standby   대기 센터 운영      중간
  Cold Site        장비 미가동         낮음

------------------------------------------------------------------------

# 4. Backup

## Full Backup

-   전체 백업
-   복구 빠름

## Incremental Backup

-   변경분만 백업
-   백업 빠름
-   복구 복잡

## Differential Backup

-   마지막 Full 이후 변경분

------------------------------------------------------------------------

# 5. Replication

## 동기(Synchronous)

-   데이터 손실 거의 없음
-   거리 제약

## 비동기(Asynchronous)

-   장거리 가능
-   약간의 데이터 손실 가능

------------------------------------------------------------------------

# 6. DR 구축 절차

``` text
업무 분석
   ↓
RTO/RPO 정의
   ↓
DR 아키텍처 설계
   ↓
백업/복제 구축
   ↓
DR 리허설
   ↓
정기 점검
```

# TOPCIT 실전 문제

## 문제1 (중 \| ★★★★★)

RPO가 의미하는 것은?

① 복구 시간 ② 데이터 손실 허용 범위 ③ 서버 수 ④ CPU 사용률

**정답: ②**

**해설:** RPO는 장애 시 허용 가능한 데이터 손실 시점을 의미한다.

------------------------------------------------------------------------

## 문제2 (상 \| ★★★★★)

서비스 중단 시간을 최소화해야 하는 금융 시스템에 가장 적합한 DR 방식은?

① Cold Site ② Active-Standby ③ Active-Active ④ Tape Backup

**정답: ③**

**해설:** Active-Active는 두 센터가 동시에 운영되어 가장 짧은 RTO를
제공한다.

------------------------------------------------------------------------

## 문제3 (상 \| ★★★★★)

동기 복제의 가장 큰 장점은?

① 장거리 복제 ② 데이터 손실 최소화 ③ 구축 비용 절감 ④ 네트워크 불필요

**정답: ②**

------------------------------------------------------------------------

## 문제4 (시나리오 \| ★★★★★)

야간 배치 시스템으로 RPO 24시간이 허용된다. 가장 경제적인 복제 방식은?

① 동기 복제 ② 비동기 복제 ③ Active-Active ④ Dual Write

**정답: ②**

**해설:** RPO 요구가 완화된 시스템은 비용 효율적인 비동기 복제가
적합하다.

------------------------------------------------------------------------

## 문제5 (시나리오 \| ★★★★★)

DR 구축 후 실제 재해 상황에서 복구가 실패했다. 가장 부족했던 활동은?

① 모의훈련 및 복구 테스트 ② CPU 증설 ③ DNS 삭제 ④ 스위치 교체

**정답: ①**

**해설:** DR은 구축보다 정기적인 복구 리허설과 검증이 중요하다.
