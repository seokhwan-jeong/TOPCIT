# 1. FinOps란?

FinOps는 Finance + DevOps의 합성어로, 클라우드 비용을 지속적으로
관리하고 최적화하는 운영 문화이다.

핵심 원칙 - 비용 가시성 - 공동 책임 - 지속적 최적화 - 데이터 기반
의사결정

# TOPCIT 실전 문제

## 문제1 (중)

FinOps의 가장 중요한 목표는?

① 서버 증설 ② 클라우드 비용 최적화 ③ DB 삭제 ④ 네트워크 변경

**정답:** ②

**해설:** FinOps는 비용과 성능의 균형을 맞추는 운영 체계이다.

------------------------------------------------------------------------

## 문제2 (상)

실제 부서별 클라우드 비용을 청구하는 방식은?

① Showback ② Chargeback ③ RAID ④ VPN

**정답:** ②

**해설:** Chargeback은 실제 비용을 각 조직에 배분한다.

------------------------------------------------------------------------

## 문제3 (상)

1년 이상 지속적으로 사용하는 VM 비용을 줄이는 가장 적절한 방법은?

① Spot Instance만 사용 ② Reserved Instance 또는 Savings Plan 구매 ③ VM
크기 무조건 증가 ④ Auto Scaling 제거

**정답:** ②

**해설:** 장기 사용이 예상되면 RI나 Savings Plan이 일반적으로 비용
절감에 유리하다.

------------------------------------------------------------------------

## 문제4 (시나리오)

월말 비용 분석이 어렵다. 가장 먼저 개선해야 할 것은?

① Tagging Strategy 수립 ② RAID 변경 ③ DNS 추가 ④ FTP 사용

**정답:** ①

**해설:** 일관된 태그가 있어야 프로젝트·부서별 비용 분석이 가능하다.

------------------------------------------------------------------------

## 문제5 (계산형)

서비스가 야간에는 거의 사용되지 않는다. 비용 절감 효과가 가장 큰 방법은?

① Auto Scaling과 스케줄 기반 VM 종료 ② CPU 업그레이드 ③ 동일 VM 추가 ④
로그 삭제

**정답:** ①

**해설:** 사용량이 적은 시간에는 리소스를 자동 축소하거나 종료하는 것이
효과적이다.
