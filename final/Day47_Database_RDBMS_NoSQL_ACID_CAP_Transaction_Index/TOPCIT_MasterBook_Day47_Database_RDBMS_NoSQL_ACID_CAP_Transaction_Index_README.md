# 2. ACID

-   Atomicity : 원자성
-   Consistency : 일관성
-   Isolation : 독립성
-   Durability : 지속성

은행 계좌 이체는 ACID가 대표적으로 요구되는 사례이다.

------------------------------------------------------------------------

# 3. CAP Theorem

분산 시스템에서는 다음 세 가지를 동시에 완벽하게 만족할 수 없다.

-   Consistency
-   Availability
-   Partition Tolerance

실무에서는 P를 전제로 C와 A의 균형을 선택한다.

------------------------------------------------------------------------

# 4. Transaction

``` text
BEGIN
 ↓
SQL 수행
 ↓
COMMIT / ROLLBACK
```

트랜잭션은 하나의 논리 작업 단위이다.

------------------------------------------------------------------------

# 5. Isolation Level

  수준               Dirty Read   Non-repeatable   Phantom
  ------------------ ------------ ---------------- ---------
  Read Uncommitted   O            O                O
  Read Committed     X            O                O
  Repeatable Read    X            X                O
  Serializable       X            X                X

------------------------------------------------------------------------

# 6. 정규화

-   1NF : 원자값
-   2NF : 부분 함수 종속 제거
-   3NF : 이행 함수 종속 제거
-   BCNF : 결정자 후보키

장점 - 중복 제거 - 데이터 일관성

단점 - JOIN 증가

------------------------------------------------------------------------

# 7. Index

인덱스는 검색 속도를 향상시키는 자료구조이다.

장점 - SELECT 성능 향상

단점 - INSERT/UPDATE/DELETE 비용 증가

# TOPCIT 실전 문제

## 문제1 (중 \| ★★★★★)

은행 계좌 이체에서 가장 중요한 특성은?

① CAP ② ACID ③ RAID ④ NAT

**정답: ②**

**해설:** 이체는 원자성과 일관성이 보장되어야 하므로 ACID가 핵심이다.

------------------------------------------------------------------------

## 문제2 (상 \| ★★★★★)

조회 성능 향상을 위한 가장 적절한 방법은?

① 트랜잭션 제거 ② 인덱스 생성 ③ COMMIT 생략 ④ PK 삭제

**정답: ②**

**해설:** 인덱스는 검색 경로를 최적화해 SELECT 성능을 높인다.

------------------------------------------------------------------------

## 문제3 (상 \| ★★★★★)

분산 시스템 이론은?

① ACID ② CAP ③ RAID ④ MVC

**정답: ②**

------------------------------------------------------------------------

## 문제4 (시나리오 \| ★★★★★)

동일 데이터가 여러 테이블에 반복 저장되어 수정 누락이 자주 발생한다.
가장 적절한 해결책은?

① 정규화 ② 인덱스 삭제 ③ COMMIT 제거 ④ 캐시 증가

**정답: ①**

**해설:** 정규화는 데이터 중복과 갱신 이상을 줄인다.

------------------------------------------------------------------------

## 문제5 (시나리오 \| ★★★★★)

INSERT 성능이 크게 저하되었다. 가장 먼저 점검할 것은?

① 과도한 인덱스 ② DNS ③ RAID0 ④ DHCP

**정답: ①**

**해설:** 인덱스가 많으면 데이터 변경 시 유지 비용이 증가한다.
