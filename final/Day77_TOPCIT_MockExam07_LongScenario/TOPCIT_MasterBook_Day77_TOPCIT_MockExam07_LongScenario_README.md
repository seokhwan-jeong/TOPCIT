# 시나리오

한 금융기관이 온프레미스 데이터센터를 Kubernetes 기반 Private Cloud로
전환한다. 요구사항은 다음과 같다.

-   GitOps 기반 배포
-   Active-Standby DR
-   AI(RAG) 상담 시스템 구축
-   Zero Trust 적용
-   Prometheus/Grafana 운영
-   DevSecOps 파이프라인 구축

아래 문제에 답하시오.

------------------------------------------------------------------------

# 문제 (20문항)

-   ① AS-IS 분석 및 요구사항 정의
-   ② DR 전환
-   ③ 운영 종료
-   ④ 모의해킹 **정답:** ①

-   ② Jenkins
-   ③ Nexus
-   ④ Maven **정답:** ①

-   ④ Wireshark **정답:** ①

-   ④ 백업 **정답:** ①

-   ② 최대 복구 시간
-   ③ 응답 시간
-   ④ CPU 사용률 **정답:** ①

13. 장기 실행 VM 비용 절감 방법은?

-   ① Reserved Instance / Savings Plan
-   ② VM 추가
-   ③ 로그 삭제
-   ④ Scale Up **정답:** ①

14. Data Lineage의 장점은?

-   ① 영향도 분석
-   ② CPU 최적화
-   ③ 메모리 압축
-   ④ DNS 관리 **정답:** ①

-   ① Never Trust, Always Verify
-   ② 내부는 항상 신뢰

-   ④ 암호화 불필요 **정답:** ①
