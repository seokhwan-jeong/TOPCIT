# 1. Enterprise Architecture(EA)

EA는 기업의 비즈니스와 IT를 정렬(Alignment)하기 위한 아키텍처 체계이다.

목표 - 비즈니스-IT 일치 - 표준화 - 투자 최적화 - 변화 대응

``` text
Business
    │
Application
    │
Data
    │
Technology
```

------------------------------------------------------------------------

# 2. TOGAF

TOGAF(Open Group Architecture Framework)는 가장 널리 사용되는 EA
프레임워크이다.

핵심 구성 - ADM(Architecture Development Method) - Content Framework -
Enterprise Continuum - Architecture Repository

ADM 단계

1.  Preliminary
2.  Architecture Vision
3.  Business Architecture
4.  Information Systems Architecture
5.  Technology Architecture
6.  Opportunities & Solutions
7.  Migration Planning
8.  Implementation Governance
9.  Architecture Change Management

------------------------------------------------------------------------

# 3. Zachman Framework

행(Row): 관점 - Planner - Owner - Designer - Builder - Subcontractor -
Enterprise

열(Column) - What - How - Where - Who - When - Why

TOGAF는 개발 방법론, Zachman은 분류 체계라는 차이가 있다.

------------------------------------------------------------------------

# 4. EA 4대 아키텍처

  구분          설명
  ------------- ------------------------
  Business      업무 프로세스
  Application   시스템 구성
  Data          데이터 구조
  Technology    HW, OS, Network, Cloud

------------------------------------------------------------------------

# 5. Architecture Governance

목적 - 표준 준수 - 설계 품질 확보 - 변경 통제 - 기술 표준 관리

주요 활동 - Architecture Review - 표준 검토 - 예외 승인

------------------------------------------------------------------------

# 6. ARB(Architecture Review Board)

역할 - 설계 검토 - 기술 표준 준수 확인 - 위험 분석 - 예외 승인

실무에서는 구축 전 설계서 심의를 수행한다.

# TOPCIT 실전 문제

## 문제1 (중)

기업의 비즈니스와 IT를 정렬하기 위한 체계는?

① EA ② RAID ③ VLAN ④ Scrum

**정답: ①**

**해설:** EA는 기업 전략과 IT를 연계하여 효율적인 아키텍처를 구축한다.

------------------------------------------------------------------------

## 문제2 (상)

TOGAF에서 아키텍처 개발 절차를 정의하는 것은?

① ADM ② CMDB ③ KPI ④ SLA

**정답: ①**

**해설:** ADM은 TOGAF의 핵심 개발 방법론이다.

------------------------------------------------------------------------

## 문제3 (상)

Zachman Framework의 특징은?

① 애자일 개발 방법 ② EA 분류 체계 ③ DB 설계 기법 ④ 테스트 자동화

**정답: ②**

------------------------------------------------------------------------

## 문제4 (시나리오)

새로운 데이터센터 구축 전 기술 표준 준수 여부를 심의하는 조직은?

① CAB ② ARB ③ SOC ④ PMO

**정답: ②**

**해설:** ARB는 아키텍처와 기술 표준을 검토하는 조직이다.

------------------------------------------------------------------------

## 문제5 (시나리오)

기존 시스템을 분석한 뒤 목표 아키텍처를 설계하는 단계는?

① TO-BE 아키텍처 수립 ② 장애 처리 ③ 백업 수행 ④ 운영 종료

**정답: ①**

**해설:** EA 프로젝트는 AS-IS 분석 후 TO-BE 아키텍처를 정의하는 것이
핵심이다.
