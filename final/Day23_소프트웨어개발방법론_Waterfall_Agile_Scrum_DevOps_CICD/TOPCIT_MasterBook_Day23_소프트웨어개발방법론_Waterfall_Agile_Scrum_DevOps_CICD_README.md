# 1. 소프트웨어 개발 방법론

개발 방법론은 프로젝트를 계획·개발·테스트·배포하는 절차와 원칙이다.

``` text
요구사항
   ↓
설계
   ↓
구현
   ↓
테스트
   ↓
배포
```

------------------------------------------------------------------------

# 2. Waterfall

단계를 순차적으로 진행하는 전통적인 개발 방식이다.

## 특징

-   요구사항이 명확할 때 적합
-   문서 중심
-   단계별 완료 후 다음 단계 진행

장점 - 관리가 쉽다. - 산출물이 명확하다.

단점 - 요구사항 변경 대응이 어렵다.

------------------------------------------------------------------------

# 3. Agile

짧은 반복 개발(Iteration)을 통해 빠르게 기능을 제공한다.

핵심 가치

-   고객과의 협업
-   변화 대응
-   동작하는 소프트웨어
-   지속적인 개선

대표 방법론

-   Scrum
-   Kanban
-   XP

------------------------------------------------------------------------

# 4. Scrum

Sprint를 반복하며 개발한다.

## 주요 역할

  역할               설명
  ------------------ -------------------
  Product Owner      요구사항 우선순위
  Scrum Master       프로세스 지원
  Development Team   개발 수행

## 주요 이벤트

-   Sprint Planning
-   Daily Scrum
-   Sprint Review
-   Sprint Retrospective

------------------------------------------------------------------------

# 5. DevOps

개발(Dev)과 운영(Ops)을 통합하여 빠르고 안정적인 서비스를 제공하는
문화와 방법론이다.

목표

-   자동화
-   협업
-   빠른 배포
-   안정성 향상

------------------------------------------------------------------------

# 6. CI/CD

## CI (Continuous Integration)

-   코드 자동 빌드
-   자동 테스트
-   코드 품질 검사

대표 도구

-   Jenkins
-   GitHub Actions
-   GitLab CI

## CD (Continuous Delivery/Deployment)

-   자동 배포
-   롤백
-   Blue-Green
-   Canary 배포

# 예상 문제

## 문제 1

요구사항 변경이 빈번한 프로젝트에 적합한 방법론은?

1.  Waterfall
2.  Agile
3.  V-Model
4.  Spiral만 가능

**정답:** 2

### 상세 해설

Agile은 반복 개발과 지속적인 피드백을 통해 요구사항 변경에 유연하게
대응한다.

------------------------------------------------------------------------

## 문제 2

Scrum에서 제품 우선순위를 관리하는 역할은?

1.  Scrum Master
2.  Product Owner
3.  Tester
4.  DBA

**정답:** 2

### 상세 해설

Product Owner는 Product Backlog를 관리하고 비즈니스 가치를 극대화한다.

------------------------------------------------------------------------

## 문제 3

CI의 주요 목적은?

1.  자동 배포만 수행
2.  코드 통합과 자동 테스트
3.  데이터 백업
4.  DB 튜닝

**정답:** 2

### 상세 해설

CI는 코드 변경 사항을 자주 통합하고 자동으로 빌드와 테스트를 수행해
품질을 유지한다.

------------------------------------------------------------------------

## 문제 4

다음 중 DevOps 도구가 아닌 것은?

1.  Jenkins
2.  GitHub Actions
3.  Argo CD
4.  Bubble Sort

**정답:** 4

### 상세 해설

Bubble Sort는 정렬 알고리즘이며 나머지는 DevOps 파이프라인에서 널리
사용하는 도구이다.

------------------------------------------------------------------------

## 문제 5

Sprint 종료 후 결과를 이해관계자에게 공유하는 이벤트는?

1.  Daily Scrum
2.  Sprint Planning
3.  Sprint Review
4.  Backlog Grooming

**정답:** 3

### 상세 해설

Sprint Review에서는 개발 결과를 시연하고 피드백을 받아 다음 Sprint에
반영한다.
