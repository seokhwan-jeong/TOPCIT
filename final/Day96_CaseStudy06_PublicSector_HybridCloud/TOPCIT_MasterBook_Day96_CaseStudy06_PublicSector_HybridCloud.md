# Day 96 - 실전 Case Study ⑥ (공공기관 클라우드 전환)

> 목표: **TOPCIT 550\~600점**

## 프로젝트 시나리오

한 공공기관이 노후 데이터센터를 하이브리드 클라우드로 전환한다.

### 요구사항

-   ISMS-P 준수
-   개인정보 암호화
-   Kubernetes 기반 업무 시스템
-   Active-Standby DR
-   GitOps / DevSecOps
-   예산 15% 절감
-   무중단 민원 서비스

당신은 인프라 PM으로 기술과 비용, 보안을 모두 고려해 의사결정을 내려야
한다.

------------------------------------------------------------------------

## 사례 분석 과제

1.  Public Cloud와 Private Cloud를 병행하는 이유를 설명하시오.
2.  Active-Standby를 선택해야 하는 상황을 설명하시오.
3.  ISMS-P 관점에서 반드시 확인해야 할 보안 항목 5가지를 작성하시오.
4.  Kubernetes 도입 시 기대 효과와 위험요인을 각각 3가지 제시하시오.
5.  프로젝트 종료 후 운영 전환 체크리스트를 작성하시오.

------------------------------------------------------------------------

# 연계 문제 (20문항)

1.  하이브리드 클라우드의 장점은? ① 유연성과 규제 대응 ② Public만 사용 ③
    Private만 사용 ④ 비용만 증가 정답:①

2.  Active-Standby의 장점은? ① 구현이 비교적 단순 ② 무조건
    Active-Active보다 빠름 ③ DR 불필요 ④ 비용 0원 정답:①

3.  ISMS-P 목적은? ① 정보보호 및 개인정보 보호 관리체계 ② DB 튜닝 ③ 백업
    ④ 성능 테스트 정답:①

4.  Kubernetes Secret 용도는? ① 민감정보 저장 ② 로그 저장 ③ 이미지 저장
    ④ 백업 정답:①

5.  ConfigMap 용도는? ① 일반 설정 관리 ② 비밀번호 저장 ③ 인증서 저장 ④
    DB 생성 정답:①

6.  GitOps 장점은? ① 변경 이력 관리와 자동 배포 ② 수동 운영 증가 ③ 로그
    삭제 ④ 성능 저하 정답:①

7.  Argo CD 역할은? ① Git 상태와 클러스터 동기화 ② 빌드 ③ 테스트 ④ 백업
    정답:①

8.  Trivy는? ① 이미지 취약점 검사 ② Dashboard ③ Tracing ④ ETL 정답:①

9.  SAST는? ① 소스코드 정적 분석 ② 실행 중 분석 ③ 백업 ④ RAID 정답:①

10. DAST는? ① 실행 중 보안 테스트 ② 정적 분석 ③ 로그 분석 ④ VPN 정답:①

11. SCA는? ① 오픈소스 취약점 분석 ② DB 암호화 ③ DNS 관리 ④ 스토리지
    정답:①

12. SBOM은? ① 공급망 보안 ② 네트워크 분리 ③ 라우팅 ④ 압축 정답:①

13. Prometheus는? ① Metrics 수집 ② 컴파일 ③ DB 생성 ④ 암호화 정답:①

14. Grafana는? ① Dashboard 시각화 ② 백업 ③ DHCP ④ RAID 정답:①

15. OpenTelemetry는? ① Distributed Tracing ② FTP ③ NAT ④ VLAN 정답:①

16. SPI\<1은? ① 일정 지연 ② 일정 앞섬 ③ 비용 절감 ④ 품질 향상 정답:①

17. CPI\>1은? ① 비용 효율 양호 ② 비용 초과 ③ 일정 지연 ④ 장애 정답:①

18. Zero Trust 핵심 원칙은? ① Never Trust, Always Verify ② 내부망은 항상
    신뢰 ③ VPN만 사용 ④ 암호화 불필요 정답:①

19. 운영 전환(Cutover) 전 반드시 수행할 것은? ① 리허설 및 롤백 계획 검증
    ② 운영 종료 ③ 로그 삭제 ④ DB 포맷 정답:①

20. PM의 최우선 역할은? ① 범위·일정·비용·품질 균형 관리 ② 직접 개발 ③
    서버 조립 ④ 코딩 정답:①
