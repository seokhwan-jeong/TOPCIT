# 1. BCP(Business Continuity Planning)

BCP는 재해 발생 시 핵심 업무를 지속하기 위한 경영 계획이다.

목표 - 업무 연속성 확보 - 고객 서비스 유지 - 재무 손실 최소화

------------------------------------------------------------------------

# 2. DRP(Disaster Recovery Planning)

DRP는 IT 시스템을 재해 이후 복구하기 위한 기술적 계획이다.

구성 - 복구 절차 - 복구 조직 - 복구 자원 - 복구 테스트

BCP가 상위 개념이며 DRP는 IT 복구를 담당한다.

------------------------------------------------------------------------

# 3. RTO / RPO

  항목   의미
  ------ -----------------------------------
  RTO    허용 가능한 최대 복구 시간
  RPO    허용 가능한 최대 데이터 손실 시점

예시 - RTO = 2시간 - RPO = 15분

------------------------------------------------------------------------

# 4. DR 아키텍처

  방식             특징
  ---------------- -------------------
  Active-Active    양 센터 동시 운영
  Active-Standby   대기 센터 운영
  Cold Site        장비 미가동
  Warm Site        일부 가동
  Hot Site         즉시 서비스 가능

------------------------------------------------------------------------

# 5. 백업 전략

3-2-1 Rule

-   데이터 3개
-   저장매체 2종
-   원격지 1개

백업 방식

-   Full
-   Incremental
-   Differential
-   Snapshot

------------------------------------------------------------------------

# 6. 복제(Replication)

동기 복제 - 데이터 손실 최소 - 짧은 거리

비동기 복제 - 장거리 가능 - 약간의 데이터 손실 허용

------------------------------------------------------------------------

# 7. DR 운영 절차

# TOPCIT 실전 문제

## 문제1

업무 연속성을 확보하기 위한 상위 계획은?

① DRP ② BCP ③ RAID ④ SLA

**정답:** ②

**해설:** BCP는 비즈니스 전체의 연속성을 보장하는 계획이며 DRP를
포함한다.

------------------------------------------------------------------------

## 문제2

RPO가 의미하는 것은?

① 최대 복구 시간 ② 최대 허용 데이터 손실 시점 ③ 네트워크 대역폭 ④ CPU
사용률

**정답:** ②

------------------------------------------------------------------------

## 문제3

데이터 손실을 최소화하는 복제 방식은?

① 비동기 복제 ② 동기 복제 ③ FTP ④ Tape

**정답:** ②

------------------------------------------------------------------------

## 문제4 (시나리오)

금융 시스템으로 데이터 손실을 거의 허용하지 않는다. 가장 적합한 구성은?

① Active-Active + 동기 복제 ② Cold Site ③ Full Backup만 수행 ④ 단일 서버

**정답:** ①

**해설:** 금융 시스템은 높은 가용성과 매우 낮은 RPO가 요구된다.

------------------------------------------------------------------------

## 문제5 (계산형)

RTO가 1시간이고 실제 복구에 80분이 걸렸다. 결과는?

① 목표 충족 ② RTO 초과 ③ RPO 초과 ④ 문제 없음

**정답:** ②

**해설:** 80분은 60분(RTO)을 초과했으므로 목표를 만족하지 못한다.
