# 1. AI Security 개요

AI 보안은 데이터, 모델, 서비스, 사용자까지 AI 생명주기 전체를 보호하는
활동이다.

``` text
Data
 ↓
Model Training
 ↓
Model Registry
 ↓
Inference API
 ↓
User
```

보호 대상

-   데이터
-   모델
-   프롬프트
-   API
-   추론 결과

------------------------------------------------------------------------

# 2. AI Governance

AI Governance는 AI의 개발·운영·폐기를 관리하는 정책 체계이다.

핵심 요소

-   정책
-   위험 관리
-   감사
-   설명 가능성
-   책임성

------------------------------------------------------------------------

# 3. Prompt Injection

Prompt Injection은 악의적인 입력으로 LLM의 동작을 변경하는 공격이다.

예시

-   시스템 프롬프트 무시
-   내부 정보 출력 유도
-   권한 우회

대응

-   입력 검증
-   출력 필터링
-   권한 분리
-   RAG 문서 제한

------------------------------------------------------------------------

# 4. Data Poisoning

학습 데이터를 조작하여 모델 성능을 저하시키는 공격이다.

영향

-   오답 증가
-   편향 발생
-   신뢰성 저하

대응

-   데이터 검증
-   출처 관리
-   품질 평가

------------------------------------------------------------------------

# 5. Model Security

주요 위협

-   Model Theft
-   Adversarial Attack
-   Membership Inference
-   Model Extraction

보호 방안

-   API 인증
-   Rate Limit
-   암호화
-   접근 제어

------------------------------------------------------------------------

# 6. Responsible AI

원칙

-   공정성(Fairness)
-   투명성(Transparency)
-   설명 가능성(Explainability)
-   책임성(Accountability)
-   개인정보 보호(Privacy)

------------------------------------------------------------------------

# TOPCIT 실전 문제

## 문제1 (중 \| ★★★★★)

LLM의 시스템 프롬프트를 우회하려는 공격은?

① SQL Injection ② Prompt Injection ③ RAID ④ XSS

**정답:** ②

**해설:** Prompt Injection은 모델의 지시를 변경하거나 내부 정보를
유출하도록 유도한다.

------------------------------------------------------------------------

## 문제2 (상 \| ★★★★★)

학습 데이터를 조작하여 모델 성능을 떨어뜨리는 공격은?

① Data Poisoning ② DDoS ③ VLAN ④ NAT

**정답:** ①

------------------------------------------------------------------------

## 문제3 (상 \| ★★★★★)

Responsible AI의 핵심 가치가 아닌 것은?

① 공정성 ② 투명성 ③ 책임성 ④ 무제한 데이터 수집

**정답:** ④

**해설:** Responsible AI는 개인정보 보호와 책임 있는 활용을 강조한다.

------------------------------------------------------------------------

## 문제4 (시나리오 \| ★★★★★)

사내 RAG 시스템에서 일반 직원이 인사 문서를 조회할 수 있었다. 가장
적절한 개선책은?

① GPU 증설 ② 문서 접근 권한 및 RBAC 적용 ③ 모델 삭제 ④ 스토리지 확장

**정답:** ②

**해설:** RAG는 검색 대상 문서에도 권한 제어가 반드시 적용되어야 한다.

------------------------------------------------------------------------

## 문제5 (시나리오 \| ★★★★★)

기업이 AI 서비스를 도입하면서 위험 관리와 감사 체계를 수립하려 한다.
가장 적합한 개념은?

① AI Governance ② RAID10 ③ ETL ④ DHCP

**정답:** ①

**해설:** AI Governance는 정책, 위험 관리, 감사 체계를 포함한다.
