# 1. 알고리즘 설계 기법

대표 설계 기법

  기법                  핵심 아이디어            대표 문제
  --------------------- ------------------------ ------------------------
  Divide & Conquer      분할 후 해결             Merge Sort, Quick Sort
  Dynamic Programming   부분 문제 결과 저장      Fibonacci, Knapsack
  Greedy                현재 최선 선택           Kruskal, Prim
  Backtracking          불가능한 경우 되돌아감   N-Queen
  Branch & Bound        최적해 탐색 + 가지치기   TSP

------------------------------------------------------------------------

# 2. Divide & Conquer

문제를 작은 문제로 나누어 해결한 후 결과를 합친다.

``` text
Problem
 ├── Sub1
 ├── Sub2
 └── Merge
```

장점 - 병렬 처리에 적합 - O(n log n) 알고리즘 구현 가능

대표 알고리즘 - Merge Sort - Quick Sort - Binary Search

------------------------------------------------------------------------

# 3. Dynamic Programming (DP)

동일한 부분 문제를 여러 번 계산하지 않도록 결과를 저장(Memoization)한다.

조건

-   Optimal Substructure
-   Overlapping Subproblem

예)

``` text
F(5)
 ↓
F(4),F(3)
 ↓
Memoization
```

------------------------------------------------------------------------

# 4. Greedy Algorithm

현재 순간의 최적 선택이 전체 최적해가 되는 경우 사용한다.

대표 문제

-   동전 거스름돈(일부 조건)
-   최소 신장 트리
-   활동 선택

장점 - 구현 간단 - 빠른 수행

주의 - 항상 최적해를 보장하지 않는다.

------------------------------------------------------------------------

# 5. Backtracking

해가 될 수 없는 경로를 즉시 포기하고 이전 단계로 되돌아간다.

``` text
선택
 ↓
가능?
 ├─ Yes → 진행
 └─ No  → Backtrack
```

대표 문제

-   N-Queen
-   Sudoku
-   순열/조합

------------------------------------------------------------------------

# 6. Branch & Bound

현재까지의 비용으로 최적해가 될 가능성이 없는 경로를 제거한다.

활용

-   TSP
-   최적화 문제
-   Integer Programming

------------------------------------------------------------------------

# 7. 알고리즘 선택 가이드

  상황             추천
  ---------------- ------------------
  정렬             Divide & Conquer
  중복 계산        DP
  지역 최적 선택   Greedy
  모든 경우 탐색   Backtracking
  최적해 탐색      Branch & Bound

# 예상 문제

## 문제 1

Merge Sort에 사용되는 설계 기법은?

1.  Greedy
2.  Divide & Conquer
3.  DP
4.  BFS

**정답:** 2

### 상세 해설

Merge Sort는 문제를 분할하고 정렬 후 병합하는 Divide & Conquer의 대표
사례이다.

------------------------------------------------------------------------

## 문제 2

Dynamic Programming의 핵심은?

1.  무조건 빠른 탐색
2.  결과 재사용(Memoization)
3.  Queue 사용
4.  Pivot 선택

**정답:** 2

### 상세 해설

동일한 부분 문제를 반복 계산하지 않도록 저장하여 시간복잡도를 줄인다.

------------------------------------------------------------------------

## 문제 3

항상 최적해를 보장하지 않는 알고리즘은?

1.  Greedy
2.  Merge Sort
3.  Binary Search
4.  BFS

**정답:** 1

### 상세 해설

Greedy는 현재 최선의 선택을 반복하므로 전체 최적해를 보장하지 않는
문제가 존재한다.

------------------------------------------------------------------------

## 문제 4

N-Queen 문제에 적합한 알고리즘은?

1.  Heap Sort
2.  Backtracking
3.  Bubble Sort
4.  Binary Search

**정답:** 2

### 상세 해설

가능성이 없는 경로를 즉시 제거하며 탐색하는 Backtracking의 대표
문제이다.

------------------------------------------------------------------------

## 문제 5

다음 중 최적화 문제 해결에 많이 사용되는 기법은?

1.  Branch & Bound
2.  Queue
3.  Stack
4.  DFS만 사용

**정답:** 1

### 상세 해설

Branch & Bound는 현재 비용을 기준으로 가지치기를 수행하여 최적해를
효율적으로 탐색한다.

------------------------------------------------------------------------
