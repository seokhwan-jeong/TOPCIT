# 1. 네트워크 보안

네트워크 보안은 외부 공격으로부터 시스템과 데이터를 보호하는 기술이다.

``` text
Internet
   │
Firewall
   │
WAF
   │
Load Balancer
   │
Web / WAS / DB
```

------------------------------------------------------------------------

# 2. Firewall

방화벽은 IP, Port, Protocol을 기준으로 트래픽을 허용하거나 차단한다.

## 종류

-   Packet Filtering
-   Stateful Firewall
-   Next Generation Firewall(NGFW)

주요 기능

-   접근제어
-   NAT
-   로그 기록

------------------------------------------------------------------------

# 3. IDS와 IPS

  항목          IDS           IPS
  ------------- ------------- -------------
  역할          탐지          탐지 + 차단
  위치          미러링 구간   Inline
  트래픽 차단   X             O

대표 제품

-   Snort
-   Suricata

------------------------------------------------------------------------

# 4. WAF

WAF(Web Application Firewall)는 웹 공격을 방어한다.

방어 대상

-   SQL Injection
-   XSS
-   CSRF
-   File Upload 공격

L7(Application Layer)에서 동작한다.

------------------------------------------------------------------------

# 5. VPN

공용 인터넷을 이용해 안전한 사설망을 구성한다.

## IPSec VPN

-   Site-to-Site
-   ESP
-   AH

## SSL VPN

-   원격 접속
-   HTTPS 기반

------------------------------------------------------------------------

# 6. Zero Trust

원칙

> Never Trust, Always Verify

핵심 요소

-   MFA
-   최소 권한
-   지속적 검증
-   Device Trust

------------------------------------------------------------------------

# 7. NAC

NAC(Network Access Control)

기능

-   단말 인증
-   보안 정책 검사
-   네트워크 접근 제어

# 예상 문제

## 문제 1

SQL Injection 방어에 가장 적합한 장비는?

1.  Router
2.  WAF
3.  Hub
4.  Switch

**정답:** 2

### 해설

WAF는 HTTP 요청을 분석하여 SQL Injection, XSS 등 웹 공격을 차단한다.

------------------------------------------------------------------------

## 문제 2

탐지와 차단을 모두 수행하는 장비는?

1.  IDS
2.  IPS
3.  Proxy
4.  NAT

**정답:** 2

### 해설

IPS는 Inline 방식으로 공격을 실시간 차단한다.

------------------------------------------------------------------------

## 문제 3

Site-to-Site VPN에서 가장 많이 사용하는 방식은?

1.  SSL VPN
2.  IPSec VPN
3.  FTP
4.  SMTP

**정답:** 2

### 해설

IPSec VPN은 지사 간 안전한 터널을 구성할 때 널리 사용된다.

------------------------------------------------------------------------

## 문제 4

Zero Trust의 핵심 원칙은?

1.  모든 내부 사용자는 신뢰한다.
2.  Never Trust, Always Verify
3.  Firewall만 설치하면 된다.
4.  VPN이 필요 없다.

**정답:** 2

### 해설

Zero Trust는 내부와 외부를 구분하지 않고 모든 접근을 지속적으로
검증한다.

------------------------------------------------------------------------

## 문제 5

AWS에서 가상 방화벽 역할을 하는 서비스는?

1.  Security Group
2.  Route53
3.  S3
4.  CloudFront

**정답:** 1

### 해설

Security Group은 인스턴스 단위의 상태 저장(Stateful) 가상 방화벽이다.

------------------------------------------------------------------------

# 오늘의 핵심 요약

-   Firewall = 접근 제어
-   IDS = 탐지
-   IPS = 탐지 + 차단
-   WAF = 웹 공격 방어
-   IPSec = Site-to-Site
-   SSL VPN = 원격 접속
-   Zero Trust = 지속 검증
