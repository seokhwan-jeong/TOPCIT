# 1. 컴퓨터 시스템 개요

컴퓨터 시스템은 **입력(Input)** 된 데이터를 **처리(Process)** 하여
**출력(Output)** 하는 시스템이다.

## 구성 요소

  구성요소   역할
  ---------- ------------------------------
  CPU        연산 및 제어
  Memory     프로그램 실행 및 데이터 저장
  Storage    영구 저장
  Input      입력 장치
  Output     출력 장치

``` text
Input
  │
  ▼
+----------------------+
|        CPU           |
| ALU | CU | Register  |
+----------+-----------+
           │
           ▼
+----------------------+
|      Main Memory     |
+----------+-----------+
           │
           ▼
+----------------------+
|    SSD / HDD         |
+----------------------+
```

## PM Note

CPU 사용률이 99%라고 해서 바로 CPU를 증설하면 안 된다.

확인 순서

``` text
CPU
 ↓
Memory
 ↓
Swap
 ↓
Disk I/O
 ↓
Network
 ↓
DB Lock
 ↓
Application Thread
```

# 2. Von Neumann Architecture

현대 컴퓨터 대부분은 Von Neumann 구조를 사용한다.

## 특징

-   프로그램과 데이터를 같은 메모리에 저장
-   CPU가 순차적으로 명령어 수행
-   구조가 단순하고 범용성이 높음

### 구조

``` text
+----------------------+
|      Memory          |
| Program + Data       |
+----------+-----------+
           ^
           |
+----------+-----------+
|         CPU          |
| ALU | CU | Register  |
+----------------------+
```

### 한계

**Von Neumann Bottleneck**

CPU보다 메모리 접근 속도가 느려 CPU가 대기하는 현상이다.

해결 기술

-   Cache Memory
-   Pipeline
-   Multi-Core

# 3. CPU 구조

  구성요소   설명
  ---------- ---------------------------
  ALU        산술 및 논리 연산
  CU         명령어 해석 및 제어
  Register   CPU 내부 초고속 저장 공간

## 명령 실행 과정

``` text
Fetch
  ↓
Decode
  ↓
Execute
  ↓
Write Back
```

-   CPU 구성 요소

# 암기 노트

-   CPU = ALU + CU + Register
-   Von Neumann = Program + Data를 같은 Memory에 저장
-   CPU 병목은 메모리 병목일 수 있다.

# 예상 문제

### 문제 1

CPU 내부 구성 요소가 아닌 것은?

1.  ALU
2.  Register
3.  SSD
4.  CU

**정답:** 3

### 문제 2

Von Neumann 구조의 특징은?

1.  프로그램과 데이터를 별도 메모리에 저장
2.  프로그램과 데이터를 같은 메모리에 저장
3.  Cache가 필수
4.  CPU가 메모리를 사용하지 않음

**정답:** 2

### 문제 3

CPU 명령 실행의 첫 번째 단계는?

1.  Decode
2.  Execute
3.  Fetch
4.  Write Back

**정답:** 3
