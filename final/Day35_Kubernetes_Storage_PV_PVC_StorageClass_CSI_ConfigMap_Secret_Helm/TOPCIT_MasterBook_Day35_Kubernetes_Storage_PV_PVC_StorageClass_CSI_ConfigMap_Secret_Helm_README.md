# 1. Kubernetes Storage 개요

컨테이너는 기본적으로 **휘발성(Ephemeral)** 저장소를 사용한다.

``` text
Pod
 │
PVC
 │
PV
 │
StorageClass
 │
Storage (NFS / EBS / Azure Disk)
```

------------------------------------------------------------------------

# 2. Persistent Volume(PV)

PV는 클러스터가 제공하는 **영구 스토리지 리소스**이다.

특징

-   Pod와 독립적
-   관리자 또는 StorageClass가 생성
-   재사용 가능

------------------------------------------------------------------------

# 3. Persistent Volume Claim(PVC)

PVC는 애플리케이션이 필요한 스토리지를 요청하는 객체이다.

예)

-   100Gi
-   ReadWriteOnce
-   SSD

PVC가 PV와 바인딩되면 Pod에서 사용할 수 있다.

------------------------------------------------------------------------

# 4. StorageClass

동적(Dynamic)으로 PV를 생성한다.

대표 예

-   AWS EBS CSI
-   Azure Disk CSI
-   NFS CSI

장점

-   운영 자동화
-   스토리지 표준화

------------------------------------------------------------------------

# 5. CSI(Container Storage Interface)

CSI는 Kubernetes와 스토리지 벤더를 연결하는 표준 인터페이스이다.

지원 예

-   NetApp
-   Dell
-   Pure Storage
-   Ceph
-   AWS EBS
-   Azure Disk

------------------------------------------------------------------------

# 6. ConfigMap vs Secret

  항목   ConfigMap   Secret
  ------ ----------- -------------------
  용도   일반 설정   민감 정보
  예     URL, PORT   Password, API Key
  저장   평문        Base64 인코딩

------------------------------------------------------------------------

# 7. Helm

Helm은 Kubernetes의 패키지 관리자이다.

구성

``` text
Chart
 │
Values.yaml
 │
Template
 │
Release
```

장점

-   재사용
-   버전관리
-   배포 자동화
-   Rollback

------------------------------------------------------------------------

# TOPCIT 실전 문제

## 문제1 (중)

애플리케이션이 스토리지를 요청하는 객체는?

① PV ② PVC ③ CSI ④ Pod

**정답:** ②

**해설:** PVC는 필요한 용량과 접근 모드를 선언하며 적절한 PV와 연결된다.

------------------------------------------------------------------------

## 문제2 (상)

StorageClass의 가장 큰 장점은?

① Pod 생성 ② 동적 PV 생성 ③ Service 생성 ④ Secret 암호화

**정답:** ②

**해설:** StorageClass는 CSI를 통해 필요한 시점에 PV를 자동 생성한다.

------------------------------------------------------------------------

## 문제3 (상)

DB 비밀번호를 저장하기 가장 적합한 리소스는?

① ConfigMap ② Secret ③ Deployment ④ Ingress

**정답:** ②

**해설:** Secret은 민감 정보를 저장하기 위한 Kubernetes 리소스이다.

------------------------------------------------------------------------

## 문제4 (상)

Helm의 역할은?

① VM 생성 ② Kubernetes 패키지 관리 ③ DNS 서버 ④ Load Balancer

**정답:** ②

**해설:** Helm은 Chart를 이용해 애플리케이션을 설치·업데이트·롤백한다.

------------------------------------------------------------------------

## 문제5 (상)

CSI의 역할은?

① CPU 스케줄링 ② 스토리지 인터페이스 표준 제공 ③ 네트워크 라우팅 ④ 로그
수집

**정답:** ②

**해설:** CSI는 다양한 스토리지 벤더와 Kubernetes를 연결하는 표준
규격이다.
