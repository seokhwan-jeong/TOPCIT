# 1. 암호학(Cryptography)

암호학은 데이터를 안전하게 보호하기 위한 기술이다.

핵심 목표(CIA)

-   Confidentiality(기밀성)
-   Integrity(무결성)
-   Availability(가용성)

------------------------------------------------------------------------

# 2. 대칭키 암호화

하나의 키로 암호화와 복호화를 수행한다.

대표 알고리즘

-   AES
-   ARIA
-   SEED
-   ChaCha20

장점 - 매우 빠름

단점 - 키 공유가 어렵다.

활용 - VPN - Disk Encryption - TLS Session Key

------------------------------------------------------------------------

# 3. 비대칭키 암호화

공개키(Public Key)와 개인키(Private Key)를 사용한다.

대표 알고리즘

-   RSA
-   ECC

장점 - 안전한 키 교환

단점 - 속도가 느림

------------------------------------------------------------------------

# 4. 대칭키 vs 비대칭키

  항목            대칭키          비대칭키
  --------------- --------------- ---------------
  키 개수         1개             2개
  속도            빠름            느림
  대표 알고리즘   AES             RSA, ECC
  활용            데이터 암호화   키 교환, 인증

------------------------------------------------------------------------

# 5. 해시(Hash)

입력을 고정 길이 값으로 변환한다.

특징

-   단방향
-   복호화 불가
-   동일 입력 → 동일 결과

대표 알고리즘

-   SHA-256
-   SHA-512
-   SHA-3

비밀번호 저장은 Salt와 함께 사용한다.

------------------------------------------------------------------------

# 6. 전자서명(Digital Signature)

전자서명은 송신자의 신원과 데이터 무결성을 검증한다.

절차

``` text
문서
 ↓
Hash
 ↓
개인키 서명
 ↓
공개키 검증
```

------------------------------------------------------------------------

# 7. PKI

PKI(Public Key Infrastructure)는 공개키 기반 인증 체계이다.

구성 요소

-   CA
-   RA
-   인증서
-   CRL

------------------------------------------------------------------------

# 8. TLS

TLS는 HTTPS의 보안 프로토콜이다.

기본 흐름

``` text
ClientHello
 ↓
ServerHello
 ↓
Certificate
 ↓
Key Exchange
 ↓
Session Key
 ↓
Encrypted Communication
```

TLS에서는 공개키로 세션키를 교환하고 실제 데이터는 AES와 같은 대칭키로
암호화한다.

# 예상 문제

## 문제 1

속도가 가장 빠른 암호화 방식은?

1.  RSA
2.  ECC
3.  AES
4.  SHA-256

**정답:** 3

### 상세 해설

AES는 대표적인 대칭키 알고리즘으로 대용량 데이터 암호화에 적합하다.

------------------------------------------------------------------------

## 문제 2

복호화가 불가능한 것은?

1.  RSA
2.  AES
3.  Hash
4.  ECC

**정답:** 3

### 상세 해설

해시는 단방향 함수이므로 원래 데이터를 복원할 수 없다.

------------------------------------------------------------------------

## 문제 3

전자서명 생성에 사용하는 키는?

1.  공개키
2.  개인키
3.  세션키
4.  대칭키

**정답:** 2

### 상세 해설

전자서명은 개인키로 생성하고 공개키로 검증한다.

------------------------------------------------------------------------

## 문제 4

TLS에서 실제 데이터 암호화에 주로 사용하는 것은?

1.  RSA
2.  AES
3.  SHA-256
4.  MD5

**정답:** 2

### 상세 해설

비대칭키는 키 교환에 사용하고 데이터는 성능이 뛰어난 AES로 암호화한다.

------------------------------------------------------------------------

## 문제 5

PKI에서 인증서를 발급하는 기관은?

1.  DNS
2.  CA
3.  DHCP
4.  NAT

**정답:** 2

### 상세 해설

CA(Certificate Authority)는 공개키 인증서를 발급하고 신뢰 체인을
구성한다.

------------------------------------------------------------------------

# 오늘의 핵심 요약

-   AES = 대칭키
-   RSA/ECC = 비대칭키
-   Hash = 단방향
-   전자서명 = 개인키 서명 + 공개키 검증
-   PKI = 인증서 기반 신뢰 체계
-   TLS = 공개키 + 대칭키 조합
