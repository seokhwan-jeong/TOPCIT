# 1. 파일시스템(File System)

파일시스템은 저장장치에 데이터를 저장하고 검색하기 위한 논리적 구조이다.

대표 파일시스템

  운영체제   파일시스템
  ---------- ------------
  Linux      ext4, XFS
  Windows    NTFS
  macOS      APFS

주요 기능

-   파일 생성/삭제
-   디렉터리 관리
-   권한 관리
-   메타데이터 관리
-   장애 복구

``` text
Application
    │
File System
    │
Block Layer
    │
SSD / HDD
```

------------------------------------------------------------------------

# 2. Inode

Linux에서 파일의 메타데이터를 저장하는 자료구조이다.

Inode에 저장되는 정보

-   파일 크기
-   소유자
-   권한
-   생성/수정 시간
-   Data Block 위치

**파일명은 Inode에 저장되지 않고 Directory Entry에 저장된다.**

실무에서는 다음 명령으로 확인한다.

``` bash
ls -li
stat filename
```

------------------------------------------------------------------------

# 3. Directory 구조

Directory는 파일명과 Inode 번호를 연결한다.

``` text
Directory
 ├── report.pdf -> inode 105
 ├── photo.jpg -> inode 211
 └── app.log   -> inode 305
```

TOPCIT에서는 "파일명은 어디에 저장되는가?"가 자주 출제된다.

정답: **Directory**

------------------------------------------------------------------------

# 4. Journaling File System

Journaling은 변경 내용을 먼저 Journal에 기록한 후 실제 데이터를 기록하는
방식이다.

장점

-   장애 복구 시간 단축
-   파일 손상 감소
-   시스템 안정성 향상

대표 예

-   ext4
-   NTFS
-   XFS

------------------------------------------------------------------------

# 5. RAID

RAID는 여러 디스크를 하나처럼 사용하는 기술이다.

  RAID     특징            장점               단점
  -------- --------------- ------------------ ------------------
  RAID0    Striping        성능 ↑             장애 복구 불가
  RAID1    Mirroring       안정성 ↑           용량 50%
  RAID5    Parity          성능+안정성        Write 성능 저하
  RAID6    Double Parity   2개 장애 허용      비용 증가
  RAID10   1+0             최고 성능/안정성   디스크 많이 필요

``` text
RAID0 : A B C D
Disk1 A C
Disk2 B D

RAID1
Disk1 A
Disk2 A
```

### 실무 선택 기준

-   DB : RAID10
-   로그 서버 : RAID1
-   일반 파일 : RAID5
-   백업 : RAID6

# 6. Disk I/O

CPU가 한가한데 시스템이 느리다면 Disk I/O 병목을 의심한다.

주요 명령

``` bash
iostat -x 1
iotop
sar -d
```

핵심 지표

  항목    의미
  ------- --------------------
  await   평균 응답 대기시간
  util    디스크 사용률
  r/s     초당 읽기
  w/s     초당 쓰기

await와 util이 모두 높다면 스토리지 병목 가능성이 높다.

------------------------------------------------------------------------

# PM Note

CPU 90%보다 Disk I/O Wait가 높은 경우가 더 치명적인 장애일 수 있다.

확인 순서

``` text
Application
   ↓
Thread Pool
   ↓
DB
   ↓
Disk I/O
   ↓
Storage
```

# 예상 문제

## 문제 1

Linux에서 파일의 메타데이터를 저장하는 것은?

1.  Directory
2.  Inode
3.  Block
4.  Cache

**정답:** 2

### 상세 해설

Inode에는 파일 크기, 권한, 소유자, 블록 위치 등이 저장된다. 파일명은
저장되지 않는다.

------------------------------------------------------------------------

## 문제 2

파일명은 어디에 저장되는가?

1.  Inode
2.  RAM
3.  Directory
4.  SSD Cache

**정답:** 3

### 상세 해설

Directory는 파일명과 Inode 번호를 연결한다. 이 개념은 TOPCIT에서 자주
출제된다.

------------------------------------------------------------------------

## 문제 3

RAID5의 특징으로 올바른 것은?

1.  미러링만 사용
2.  Parity를 사용한다.
3.  장애 복구 불가
4.  디스크 1개만 사용

**정답:** 2

### 상세 해설

RAID5는 Striping과 Parity를 함께 사용하여 성능과 안정성을 균형 있게
제공한다.

------------------------------------------------------------------------

## 문제 4

Disk I/O 병목 여부를 확인하기 적절한 Linux 명령은?

1.  ping
2.  iostat
3.  nslookup
4.  arp

**정답:** 2

### 상세 해설

iostat는 디스크 I/O 사용률, await, util 등을 제공하여 저장장치 병목
분석에 가장 많이 사용된다.
