# 1. DevSecOps

DevSecOps는 개발(Dev), 운영(Ops), 보안(Security)을 소프트웨어 개발
생명주기에 통합하는 방법론이다.

``` text
Plan
 ↓
Code
 ↓
Build
 ↓
Test
 ↓
Security Scan
 ↓
Deploy
 ↓
Monitor
```

핵심 목표 - 보안 자동화 - 조기 취약점 발견 - 지속적 보안 검증

------------------------------------------------------------------------

# 2. Secure SDLC

  단계       보안 활동
  ---------- ------------------
  요구사항   보안 요구 분석
  설계       위협 모델링
  구현       Secure Coding
  테스트     취약점 점검
  배포       보안 검증
  운영       모니터링 및 패치

# 3. SAST / DAST / SCA

  기술   검사 대상              실행 시점
  ------ ---------------------- -----------
  SAST   소스코드               개발
  DAST   실행 중 애플리케이션   테스트
  SCA    오픈소스 라이브러리    개발/빌드

대표 도구 - SonarQube - Checkmarx - OWASP ZAP - Trivy - Snyk

------------------------------------------------------------------------

# 4. SBOM

SBOM(Software Bill of Materials)은 소프트웨어를 구성하는 모든
라이브러리와 버전 목록이다.

장점 - 취약점 추적 - 라이선스 관리 - 공급망 보안 강화

# TOPCIT 실전 문제

## 문제1 (중 \| ★★★★★)

소스코드를 분석하여 취약점을 찾는 방법은?

① DAST ② SAST ③ RAID ④ IDS

**정답:** ②

**해설:** SAST는 실행 전 소스코드를 분석해 보안 취약점을 찾는다.

------------------------------------------------------------------------

## 문제2 (상 \| ★★★★★)

실행 중인 웹 애플리케이션을 대상으로 취약점을 점검하는 방법은?

① SAST ② DAST ③ SBOM ④ ETL

**정답:** ②

**해설:** DAST는 실행 중인 서비스를 대상으로 동적 분석을 수행한다.

------------------------------------------------------------------------

## 문제3 (상 \| ★★★★★)

오픈소스 라이브러리 취약점을 분석하는 기술은?

① SCA ② RAID5 ③ Kafka ④ DHCP

**정답:** ①

------------------------------------------------------------------------

## 문제4 (시나리오 \| ★★★★★)

Log4Shell과 같은 오픈소스 취약점의 영향을 빠르게 파악하기 위해 가장
필요한 것은?

① SBOM ② DNS 변경 ③ DB 튜닝 ④ VLAN

**정답:** ①

**해설:** SBOM을 통해 영향을 받는 라이브러리와 버전을 즉시 확인할 수
있다.

------------------------------------------------------------------------

## 문제5 (시나리오 \| ★★★★★)

CI/CD에서 컨테이너 이미지의 취약점을 자동 점검하려면?

① Trivy ② FTP ③ RAID10 ④ SMTP

**정답:** ①

**해설:** Trivy는 이미지와 파일시스템의 취약점을 자동으로 검사하는 대표
도구이다.
