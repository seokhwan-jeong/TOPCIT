# 1. ITSM

ITSM(IT Service Management)은 IT 서비스를 체계적으로 설계·운영·개선하는
관리 체계이다.

목표 - 서비스 품질 향상 - 안정적 운영 - 고객 만족

------------------------------------------------------------------------

# 2. ITIL v4

Service Value System(SVS)

``` text
Demand
  ↓
Service Value Chain
  ↓
Value
```

핵심 활동 - Plan - Improve - Engage - Design & Transition -
Obtain/Build - Deliver & Support

------------------------------------------------------------------------

# 3. Incident / Problem / Change

  프로세스   목적
  ---------- ------------------
  Incident   서비스 신속 복구
  Problem    근본 원인 제거
  Change     변경 통제

예시 - 서버 다운 → Incident - 반복 다운 원인 분석 → Problem - 패치 적용
→ Change

------------------------------------------------------------------------

# 4. CMDB

CMDB(Configuration Management Database)는 IT 자산과 구성항목(CI)의
관계를 관리한다.

구성 - 서버 - 네트워크 - DB - 애플리케이션 - 관계 정보

장점 - 영향도 분석 - 장애 추적 - 변경 관리

------------------------------------------------------------------------

# 5. SLA / SLO / KPI

  항목   의미
  ------ ----------------------
  SLA    고객과의 서비스 계약
  SLO    내부 목표 수준
  KPI    성과 측정 지표

------------------------------------------------------------------------

# 6. Change Management

절차

요청 → 영향도 분석 → 승인(CAB) → 구현 → 검증 → 종료

주요 원칙 - 위험 최소화 - 사전 승인 - 롤백 계획

# TOPCIT 실전 문제

## 문제1 (중 \| ★★★★★)

서비스를 가능한 빨리 복구하는 프로세스는?

① Problem Management ② Incident Management ③ Change Management ④
Capacity Management

**정답:** ②

**해설:** Incident Management의 목표는 서비스 복구 시간을 최소화하는
것이다.

------------------------------------------------------------------------

## 문제2 (상 \| ★★★★★)

반복 장애의 근본 원인을 제거하는 프로세스는?

① Incident ② Problem ③ Release ④ Asset

**정답:** ②

**해설:** Problem Management는 Root Cause Analysis(RCA)를 수행한다.

------------------------------------------------------------------------

## 문제3 (상 \| ★★★★★)

변경 작업의 영향도를 분석하는 데 가장 중요한 정보는?

① RAID 구성 ② CMDB ③ GPU ④ DNS

**정답:** ②

------------------------------------------------------------------------

## 문제4 (시나리오 \| ★★★★★)

운영 중 데이터베이스 패치를 적용하려 한다. 가장 먼저 수행해야 할 절차는?

① 즉시 적용 ② Change 승인 및 영향도 분석 ③ 서버 삭제 ④ 백업 생략

**정답:** ②

**해설:** 변경 관리는 승인과 영향도 분석 후 수행해야 한다.

------------------------------------------------------------------------

## 문제5 (시나리오 \| ★★★★★)

고객 계약에 '월 가용성 99.9%'가 명시되어 있다. 이것은?

① KPI ② SLA ③ SLO ④ OLA

**정답:** ②

**해설:** SLA는 고객과 합의한 서비스 수준 계약이다.
