# Day 92 - 실전 Case Study ② (장애 분석 · RCA · Kubernetes 운영)

> 목표: **TOPCIT 550\~600점**

## 프로젝트 시나리오

전자정부 서비스가 Kubernetes 기반으로 운영 중이다.

### 장애 상황

-   오전 09:10부터 응답시간이 5초 이상으로 증가
-   일부 Pod는 CrashLoopBackOff
-   Prometheus에서 CPU는 정상이나 메모리 사용량 급증
-   Ingress 5xx 오류 증가
-   GitOps 배포 직후 장애 발생
-   AI(RAG) 검색 응답도 느려짐

당신은 **인프라 PM**으로서 원인 분석과 복구 계획을 수립해야 한다.

------------------------------------------------------------------------

# 분석 과제

1.  장애의 우선 원인을 3가지 가설로 제시하시오.
2.  가장 먼저 확인해야 할 로그와 메트릭은 무엇인가?
3.  즉시 조치와 영구 조치를 구분하여 작성하시오.
4.  재발 방지를 위한 RCA 항목을 작성하시오.
5.  GitOps 환경에서 안전한 롤백 절차를 설명하시오.
