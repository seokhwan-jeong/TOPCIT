# 2. JOIN

  종류              특징          사용 예
  ----------------- ------------- ------------------
  INNER JOIN        공통 데이터   주문 + 고객
  LEFT JOIN         왼쪽 전체     고객 + 주문 여부
  RIGHT JOIN        오른쪽 전체   참조 데이터
  FULL OUTER JOIN   양쪽 전체     데이터 비교
  CROSS JOIN        모든 조합     테스트

# 4. GROUP BY / HAVING

GROUP BY는 데이터를 그룹화한다.

HAVING은 그룹화 결과를 필터링한다.

``` sql
SELECT dept, COUNT(*)
FROM employee
GROUP BY dept
HAVING COUNT(*) > 10;
```

# TOPCIT 실전 문제

## 문제1 (중 \| ★★★★★)

GROUP BY 이후 조건을 지정하는 절은?

① WHERE ② ORDER BY ③ HAVING ④ LIMIT

**정답:** ③

**해설:** WHERE는 그룹화 이전, HAVING은 그룹화 이후 조건을 적용한다.

------------------------------------------------------------------------

## 문제2 (상 \| ★★★★★)

고객 전체를 조회하고 주문이 없는 고객도 함께 표시하려면?

① INNER JOIN ② LEFT JOIN ③ CROSS JOIN ④ SELF JOIN

**정답:** ②

**해설:** LEFT JOIN은 왼쪽 테이블의 모든 데이터를 유지한다.

------------------------------------------------------------------------

## 문제3 (상 \| ★★★★★)

SQL 실행 계획을 생성하는 핵심 구성 요소는?

① Parser ② Optimizer ③ Trigger ④ View

**정답:** ②

**해설:** Optimizer는 비용 기반으로 최적의 실행 계획을 선택한다.

------------------------------------------------------------------------

## 문제4 (시나리오 \| ★★★★★)

INSERT 시마다 감사 로그를 자동 저장해야 한다. 가장 적합한 기능은?

① View ② Trigger ③ GROUP BY ④ Index

**정답:** ②

**해설:** Trigger는 INSERT, UPDATE, DELETE 이벤트에 자동 실행된다.

------------------------------------------------------------------------

## 문제5 (시나리오 \| ★★★★★)

조회 속도가 매우 느린 SQL의 첫 번째 점검 항목은?

① EXPLAIN 실행 계획 ② DB 재설치 ③ Trigger 삭제 ④ PK 제거

**정답:** ①

**해설:** 실행 계획을 확인하면 Full Scan, Join 방식, 인덱스 사용 여부를
분석할 수 있다.
