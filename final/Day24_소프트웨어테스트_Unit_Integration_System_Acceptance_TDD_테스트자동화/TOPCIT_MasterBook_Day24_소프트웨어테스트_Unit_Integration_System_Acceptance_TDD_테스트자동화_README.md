# 1. 소프트웨어 테스트란?

소프트웨어 테스트는 결함(Bug)을 발견하고 품질을 확보하기 위한 검증
활동이다.

``` text
요구사항
   ↓
개발
   ↓
테스트
   ↓
배포
```

## 테스트 목적

-   품질 향상
-   결함 조기 발견
-   유지보수 비용 절감
-   고객 만족도 향상

------------------------------------------------------------------------

# 2. 테스트 레벨

  테스트             목적                 수행 주체
  ------------------ -------------------- -------------
  Unit Test          함수/클래스 검증     개발자
  Integration Test   모듈 간 연동         개발자/QA
  System Test        전체 시스템 검증     QA
  Acceptance Test    고객 요구사항 검증   고객/사용자

------------------------------------------------------------------------

# 3. Unit Test

가장 작은 단위(메서드, 함수)를 검증한다.

대표 도구

-   JUnit
-   NUnit
-   pytest

장점

-   버그 조기 발견
-   리팩터링 안전성 확보

------------------------------------------------------------------------

# 4. Integration Test

모듈 간 인터페이스와 데이터 흐름을 검증한다.

예시

``` text
Web
 ↓
API
 ↓
DB
```

확인 사항

-   API 호출
-   DB 연동
-   메시지 큐
-   외부 시스템 연계

------------------------------------------------------------------------

# 5. System Test

운영 환경과 유사한 환경에서 전체 기능을 검증한다.

주요 항목

-   기능 테스트
-   성능 테스트
-   보안 테스트
-   장애 복구 테스트

------------------------------------------------------------------------

# 6. Acceptance Test(UAT)

사용자 관점에서 요구사항 충족 여부를 확인한다.

대표 시나리오

-   회원가입
-   로그인
-   주문
-   결제

------------------------------------------------------------------------

# 7. TDD(Test Driven Development)

절차

``` text
Fail
 ↓
Code
 ↓
Pass
 ↓
Refactoring
```

장점

-   높은 코드 품질
-   테스트 가능한 설계
-   유지보수 향상

------------------------------------------------------------------------

# 8. 테스트 자동화

대표 도구

  분야   도구
  ------ ----------------------
  Unit   JUnit, pytest
  UI     Selenium, Playwright
  API    Postman, Newman
  성능   JMeter, k6

자동화 효과

-   반복 작업 감소
-   빠른 회귀 테스트
-   CI/CD 연계

------------------------------------------------------------------------

# 예상 문제

## 문제 1

가장 작은 단위의 테스트는?

1.  Integration Test
2.  Unit Test
3.  System Test
4.  Acceptance Test

**정답:** 2

### 상세 해설

Unit Test는 함수나 클래스와 같은 최소 단위를 검증하며 개발자가 가장 먼저
수행한다.

------------------------------------------------------------------------

## 문제 2

사용자 요구사항 충족 여부를 확인하는 테스트는?

1.  Unit Test
2.  Acceptance Test
3.  Load Test
4.  Smoke Test

**정답:** 2

### 상세 해설

Acceptance Test(UAT)는 고객 또는 사용자가 요구사항을 만족하는지 검증하는
마지막 단계이다.

------------------------------------------------------------------------

## 문제 3

TDD의 올바른 순서는?

1.  Code → Test → Deploy
2.  Fail → Code → Pass → Refactoring
3.  Build → Test → Release
4.  Design → Deploy → Test

**정답:** 2

### 상세 해설

TDD는 실패하는 테스트를 먼저 작성하고, 이를 통과하는 코드를 구현한 뒤
리팩터링을 수행한다.

------------------------------------------------------------------------

## 문제 4

UI 자동화 테스트 도구는?

1.  JMeter
2.  Selenium
3.  Git
4.  Maven

**정답:** 2

### 상세 해설

Selenium은 브라우저 기반 UI 테스트 자동화의 대표 도구이다.

------------------------------------------------------------------------

## 문제 5

CI 파이프라인에서 가장 먼저 자동 수행하는 작업은?

1.  운영 배포
2.  코드 빌드 및 테스트
3.  장애 복구
4.  백업

**정답:** 2

### 상세 해설

CI는 코드 변경 시 자동 빌드와 테스트를 수행하여 품질을 빠르게 검증한다.

------------------------------------------------------------------------

# 오늘의 핵심 요약

-   Unit = 함수/클래스
-   Integration = 모듈 연동
-   System = 전체 시스템
-   Acceptance = 사용자 검증
-   TDD = Fail → Code → Pass → Refactoring
-   자동화 = CI/CD 핵심
