# 1. 정렬(Sorting)이란?

정렬은 데이터를 일정한 기준(오름차순, 내림차순 등)에 맞게 재배치하는
과정이다.

``` text
[8][3][5][1]
      │
 Sorting
      │
[1][3][5][8]
```

대표 활용

-   데이터 조회
-   이진 탐색
-   보고서 출력
-   검색 최적화

------------------------------------------------------------------------

# 2. Bubble Sort

인접한 두 원소를 비교하며 큰 값을 뒤로 이동시킨다.

``` text
8 3 5 1
↓
3 5 1 8
```

-   평균: O(n²)
-   구현이 쉬움
-   대용량 데이터에는 부적합

------------------------------------------------------------------------

# 3. Selection Sort

가장 작은 값을 선택하여 앞쪽으로 이동한다.

장점 - 교환 횟수가 적음

단점 - 비교 횟수가 많음 - O(n²)

------------------------------------------------------------------------

# 4. Insertion Sort

이미 정렬된 영역에 데이터를 삽입한다.

장점

-   거의 정렬된 데이터에서 매우 빠름

단점

-   평균 O(n²)

------------------------------------------------------------------------

# 5. Merge Sort

분할 정복(Divide & Conquer) 알고리즘이다.

``` text
8 3 5 1
 ├─ 8 3
 └─ 5 1
   ...
 Merge
```

-   평균 O(n log n)
-   안정 정렬(Stable)
-   추가 메모리 필요

------------------------------------------------------------------------

# 6. Quick Sort

Pivot을 기준으로 작은 값과 큰 값을 분리한다.

장점

-   평균 O(n log n)
-   실제로 매우 빠름

단점

-   최악 O(n²)
-   Pivot 선택 중요

------------------------------------------------------------------------

# 7. Heap Sort

Heap 자료구조를 이용한다.

-   평균 O(n log n)
-   추가 메모리 거의 없음
-   우선순위 큐 구현에 활용

------------------------------------------------------------------------

# 8. 시간복잡도 비교

  알고리즘            평균         최악 안정 정렬
  ----------- ------------ ------------ -----------
  Bubble             O(n²)        O(n²) O
  Selection          O(n²)        O(n²) X
  Insertion          O(n²)        O(n²) O
  Merge         O(n log n)   O(n log n) O
  Quick         O(n log n)        O(n²) X
  Heap          O(n log n)   O(n log n) X

------------------------------------------------------------------------

# 실무 사례

-   Java Collections.sort() : TimSort(상황에 따라 Merge/Insertion)
-   C++ std::sort() : Introsort(Quick+Heap)
-   데이터베이스 ORDER BY : Optimizer가 내부 정렬 전략 선택
-   Hadoop/Spark : Merge 기반 정렬 활용

# PM Note

대용량 데이터에서는 O(n²) 정렬을 피하는 것이 일반적이다.

``` text
100건 → Bubble 가능
10만건 → Merge / Quick 권장
1000만건 → 분산 정렬 고려
```

# 예상 문제

## 문제 1

평균 시간복잡도가 O(n log n)인 알고리즘이 아닌 것은?

1.  Merge Sort
2.  Quick Sort
3.  Heap Sort
4.  Bubble Sort

**정답:** 4

### 상세 해설

Bubble Sort는 평균과 최악 모두 O(n²)이다.

------------------------------------------------------------------------

## 문제 2

거의 정렬된 데이터에서 가장 효율적인 알고리즘은?

1.  Bubble Sort
2.  Insertion Sort
3.  Selection Sort
4.  Heap Sort

**정답:** 2

### 상세 해설

Insertion Sort는 이동 횟수가 적어 거의 정렬된 데이터에서 매우
효율적이다.

------------------------------------------------------------------------

## 문제 3

Quick Sort의 핵심 요소는?

1.  Queue
2.  Pivot
3.  Stack
4.  Hash

**정답:** 2

### 상세 해설

Pivot을 기준으로 데이터를 분할하는 것이 Quick Sort의 핵심 원리이다.

------------------------------------------------------------------------

## 문제 4

안정 정렬(Stable Sort)에 해당하는 것은?

1.  Merge Sort
2.  Heap Sort
3.  Selection Sort
4.  Quick Sort

**정답:** 1

### 상세 해설

Merge Sort는 동일한 값의 상대적 순서를 유지하는 안정 정렬이다.

------------------------------------------------------------------------

## 문제 5

Heap Sort가 사용하는 자료구조는?

1.  Linked List
2.  Heap
3.  Graph
4.  Queue

**정답:** 2

### 상세 해설

Heap Sort는 완전 이진트리 기반 Heap 자료구조를 이용하여 정렬을 수행한다.

------------------------------------------------------------------------

# 오늘의 핵심 요약

-   Bubble = 구현 쉬움
-   Selection = 최소값 선택
-   Insertion = 거의 정렬된 데이터에 강함
-   Merge = 안정 정렬, O(n log n)
-   Quick = 평균 가장 빠름
-   Heap = Heap 기반 O(n log n)
