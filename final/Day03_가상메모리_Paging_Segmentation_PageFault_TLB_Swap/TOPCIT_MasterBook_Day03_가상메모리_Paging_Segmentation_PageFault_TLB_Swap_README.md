# 1. 가상 메모리(Virtual Memory)

가상 메모리는 **물리 메모리(RAM)가 부족하더라도 더 큰 메모리를 사용하는
것처럼 보이게 하는 운영체제 기술**이다.

## 장점

-   프로세스마다 독립적인 주소 공간 제공
-   메모리 효율 향상
-   프로그램 크기가 RAM보다 커도 실행 가능

``` text
Application
    │
Virtual Address
    │
MMU
    │
Physical Address
    │
RAM
```

------------------------------------------------------------------------

# 2. Paging

Paging은 메모리를 **고정된 크기의 Page**로 나누어 관리하는 기법이다.

  용어         설명
  ------------ ------------------
  Page         가상 메모리 블록
  Frame        물리 메모리 블록
  Page Table   주소 변환 정보

### 장점

-   External Fragmentation 제거

### 단점

-   Internal Fragmentation 발생 가능

------------------------------------------------------------------------

# 3. Segmentation

Segmentation은 프로그램을 **논리적 단위(Code, Data, Stack)** 로 나누는
방식이다.

``` text
Program
 ├── Code
 ├── Data
 └── Stack
```

### Paging vs Segmentation

  항목     Paging       Segmentation
  -------- ------------ ---------------
  기준     고정 크기    논리 단위
  단편화   Internal     External
  관리     Page Table   Segment Table

------------------------------------------------------------------------

# 4. Page Fault

CPU가 접근하려는 Page가 RAM에 없을 때 발생한다.

``` text
CPU Access
     │
Page 존재?
 ├── Yes → 실행
 └── No
       │
Page Fault
       │
Disk Read
       │
RAM 적재
       │
재실행
```

### Major Page Fault

Disk 접근 필요

### Minor Page Fault

이미 메모리에 존재하여 빠르게 처리

------------------------------------------------------------------------

# 5. TLB (Translation Lookaside Buffer)

TLB는 Page Table 접근을 줄이기 위한 고속 Cache이다.

``` text
CPU
 │
TLB
 │
Page Table
 │
RAM
```

### TLB Hit

주소 변환 즉시 완료

### TLB Miss

Page Table 접근 필요

------------------------------------------------------------------------

# 6. Swap

RAM이 부족하면 일부 메모리를 Disk로 이동한다.

이를 Swap이라고 한다.

``` text
RAM 부족
   │
Swap Out
   │
Disk
```

### 실무 사례

Swap 사용률이 높으면 CPU 사용률도 높아질 수 있다.

이유는 Disk I/O 대기가 증가하기 때문이다.

------------------------------------------------------------------------

# PM Note

CPU 사용률이 높은데 원인을 찾지 못했다면 반드시 Swap을 확인한다.

``` bash
free -h
vmstat 1
top
```

Swap In(si), Swap Out(so)이 증가하면 메모리 부족을 의심한다.

------------------------------------------------------------------------

# 실무 사례

## Kubernetes

Pod가 Memory Limit을 초과하면 OOMKilled가 발생할 수 있다.

Swap을 사용하는 환경에서는 성능 저하가 심해질 수 있으므로 운영
환경에서는 일반적으로 Swap을 비활성화한다.

# 예상 문제

## 문제 1

Virtual Memory의 가장 큰 목적은?

1.  CPU 속도 향상
2.  RAM보다 큰 메모리를 사용하는 것처럼 제공
3.  SSD 속도 향상
4.  Cache 증가

**정답:** 2

### 상세 해설

가상 메모리는 실제 RAM보다 큰 주소 공간을 제공하여 프로그램을 효율적으로
실행할 수 있게 한다.

------------------------------------------------------------------------

## 문제 2

Paging의 장점은?

1.  External Fragmentation 제거
2.  CPU 사용률 감소
3.  Disk 제거
4.  Register 증가

**정답:** 1

### 상세 해설

Paging은 메모리를 고정 크기로 나누므로 외부 단편화를 제거할 수 있다.

------------------------------------------------------------------------

## 문제 3

TLB의 역할은?

1.  CPU 온도 측정
2.  주소 변환 속도 향상
3.  RAID 관리
4.  DNS Cache

**정답:** 2

### 상세 해설

TLB는 최근 주소 변환 정보를 저장하여 Page Table 접근 횟수를 줄인다.

------------------------------------------------------------------------

## 문제 4

Page Fault가 발생하면 가장 먼저 수행되는 작업은?

1.  서버 재부팅
2.  필요한 Page를 Disk에서 RAM으로 적재
3.  Cache 삭제
4.  CPU 증설

**정답:** 2

### 상세 해설

RAM에 필요한 Page가 없기 때문에 운영체제가 Disk에서 Page를 읽어 메모리에
적재한다.

------------------------------------------------------------------------

## 문제 5

Swap 사용률이 지속적으로 높다면 가장 적절한 판단은?

1.  CPU가 부족하다.
2.  메모리가 부족하거나 메모리 사용량이 과다하다.
3.  DNS 오류이다.
4.  RAID 문제이다.

**정답:** 2

### 상세 해설

Swap은 RAM이 부족할 때 사용되는 공간이다. Swap이 계속 증가하면 메모리
부족 또는 Memory Leak을 의심해야 한다.

------------------------------------------------------------------------

# 기술면접

### Q. Paging과 Segmentation의 차이를 설명해 보세요.

**모범답변**

Paging은 고정 크기 Page 단위로 메모리를 관리하며 외부 단편화를 제거한다.
Segmentation은 Code, Data, Stack과 같은 논리 단위로 관리하며 프로그램
구조를 반영한다.

------------------------------------------------------------------------

# 암기노트

-   Virtual Memory = RAM보다 큰 주소 공간
-   Paging = 고정 크기
-   Segmentation = 논리 단위
-   Page Fault = Disk 접근
-   TLB = 주소 변환 Cache
-   Swap = RAM 부족 시 Disk 사용
