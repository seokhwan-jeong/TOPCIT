# 1. 자료구조(Data Structure)

자료구조는 데이터를 효율적으로 저장하고 탐색하기 위한 방법이다.

``` text
데이터
  │
자료구조
  │
알고리즘
  │
결과
```

대표 자료구조

-   Array
-   Linked List
-   Stack
-   Queue
-   Tree
-   Hash Table

------------------------------------------------------------------------

# 2. Big-O 시간복잡도

  복잡도       의미        예시
  ------------ ----------- ---------------
  O(1)         상수 시간   Hash 조회
  O(log n)     로그 시간   Binary Search
  O(n)         선형        순차 탐색
  O(n log n)   정렬        Merge Sort
  O(n²)        이중 반복   Bubble Sort

------------------------------------------------------------------------

# 3. Array

특징

-   연속된 메모리
-   Index 접근 O(1)

장점 - 조회 빠름

단점 - 중간 삽입/삭제 O(n)

활용 - 테이블 - 이미지 - 행렬

------------------------------------------------------------------------

# 4. Linked List

노드가 포인터로 연결된다.

``` text
[Data|Next] -> [Data|Next] -> NULL
```

장점 - 삽입/삭제 빠름

단점 - Index 접근 느림

------------------------------------------------------------------------

# 5. Stack

LIFO(Last In First Out)

``` text
Push
Push
Push
 ↓
Pop
```

활용 - 함수 호출 - DFS - Undo

------------------------------------------------------------------------

# 6. Queue

FIFO(First In First Out)

``` text
Enqueue
Enqueue
 ↓
Dequeue
```

활용 - 프린터 - 메시지 큐 - BFS

------------------------------------------------------------------------

# 7. Tree

계층 구조를 표현한다.

``` text
      Root
     /    \
   A        B
  / \      /
 C   D    E
```

종류

-   Binary Tree
-   BST
-   AVL
-   B-Tree

실무 - 파일시스템 - DB Index(B+Tree)

------------------------------------------------------------------------

# 8. Hash Table

Key를 Hash Function으로 계산하여 저장한다.

장점

-   평균 O(1)

단점

-   Collision 발생 가능

충돌 해결

-   Chaining
-   Open Addressing

실무

-   HashMap
-   Redis Dictionary

------------------------------------------------------------------------

# 9. 자료구조 비교

  자료구조          조회     삽입     삭제
  ------------- -------- -------- --------
  Array             O(1)     O(n)     O(n)
  Linked List       O(n)     O(1)     O(1)
  Stack             O(n)     O(1)     O(1)
  Queue             O(n)     O(1)     O(1)
  Hash Table      O(1)\*   O(1)\*   O(1)\*

\*평균 기준

------------------------------------------------------------------------

# 실무 사례

-   Java HashMap → Hash Table
-   Linux Ready Queue → Queue
-   브라우저 뒤로가기 → Stack
-   Kubernetes Work Queue → Queue
-   Oracle Index → B+Tree
-   Redis Dictionary → Hash

# PM Note

자료구조를 잘못 선택하면 하드웨어를 증설해도 성능 문제가 해결되지 않을
수 있다.

예)

-   검색이 많은 시스템 → Hash
-   정렬이 많은 시스템 → Tree
-   순차 처리 → Queue

# 예상 문제

## 문제 1

Index 접근이 가장 빠른 자료구조는?

1.  Linked List
2.  Queue
3.  Array
4.  Tree

**정답:** 3

### 상세 해설

Array는 연속된 메모리 구조를 사용하므로 인덱스로 O(1)에 접근할 수 있다.

------------------------------------------------------------------------

## 문제 2

LIFO 구조는?

1.  Queue
2.  Stack
3.  Tree
4.  Graph

**정답:** 2

### 상세 해설

Stack은 마지막에 저장한 데이터를 먼저 꺼내는 LIFO 구조이다.

------------------------------------------------------------------------

## 문제 3

Hash Table의 평균 조회 시간복잡도는?

1.  O(n²)
2.  O(log n)
3.  O(1)
4.  O(n)

**정답:** 3

### 상세 해설

충돌이 심하지 않다는 가정에서 평균적으로 O(1)의 조회 성능을 제공한다.

------------------------------------------------------------------------

## 문제 4

B+Tree가 가장 많이 사용되는 곳은?

1.  CPU Register
2.  DB Index
3.  DNS Cache
4.  RAID

**정답:** 2

### 상세 해설

관계형 데이터베이스는 디스크 I/O를 최소화하기 위해 B+Tree 기반 인덱스를
사용한다.

------------------------------------------------------------------------

## 문제 5

프린터 대기열에 가장 적합한 자료구조는?

1.  Stack
2.  Queue
3.  Tree
4.  Array

**정답:** 2

### 상세 해설

먼저 요청한 작업을 먼저 처리해야 하므로 FIFO 구조인 Queue가 적합하다.

------------------------------------------------------------------------
